# IT상상공방 라인 검증 가이드

기반공사가 끝난 공방이 **고객 요청 → 납품**까지 끊김 없이 도는지 확인하는 절차.

---

## 0. 사전 준비 (중요)

1. **반드시 `IT_make_some` 폴더에서 Claude Code 세션을 새로 연다.**
   - `.claude/`(에이전트·커맨드·규칙·훅)와 `CLAUDE.md`는 그 폴더에서 세션을 열어야 로드된다.
   - best-practice 레포 세션에서는 `/신규프로젝트` 등이 인식되지 않는다.
2. GitHub 계정 확인: `gh api user --jq .login` → `LeeSiHyeon711` 이어야 한다.
3. 테스트는 **작은 정적 산출물**로 한다 (예: "개인용 D-Day 카운터 웹페이지 한 장").

---

## 1. 끝까지 한 바퀴 (해피 패스)

| 입력 | 기대 결과 (이게 나오면 통과) |
|------|------------------------------|
| `/신규프로젝트 테스트` + 요청 입력 | `projects/테스트/` 에 7개 폴더 생성, `01-상담/요구사항-정의서.md` 작성 후 정지 |
| `/다음단계 테스트` | `02-기획/PRD.md` (MVP 스코프 컷 포함) |
| `/다음단계 테스트` | `03-설계/설계서.md` (형태=웹 정적, **작업 분해 목록** 포함) |
| `/다음단계 테스트` | GitHub repo `테스트` 생성, 이슈 등록, `04-깃허브/이슈등록내역.md` 에 **repo URL** |
| `/다음단계 테스트` (반복) | 이슈 1개씩 구현 → close, "남은 이슈 N개" 안내. N=0 될 때까지 반복 |
| `/다음단계 테스트` | `06-검수/검수리포트.md` (통과/보류 판정) |
| `/다음단계 테스트` | `07-납품/실행안내.md` + `고객전달문.md` |

각 단계 뒤 **항상 정지**하는지 확인한다(휴먼 게이트). 두 단계가 한 번에 진행되면 결함.

---

## 2. 4층 방어 검증

- **범위 격리**: 각 단계 산출물이 자기 폴더에만 생겼는지 확인. 예) 기획자가 `03-설계/`를 건드리면 결함.
- **권한 훅(핵심)**: 개발(5단계) 중 builder가 `git push`나 `gh`를 시도하면 `⛔ ... 권한이 없습니다` 로 차단돼야 한다.
  - 확인법: builder 작업 로그에 push/gh 호출이 없는지 + 있었다면 차단됐는지 본다.
  - 이슈 close는 항상 **repo-manager**가 했는지 `이슈등록내역.md` 동기화 로그로 확인.
- **deny 규칙**: `git push --force` 나 `gh repo delete` 는 어떤 경우에도 차단되는지(필요시 수동 테스트).

---

## 3. ★ 끊김 복원력 검증 (이 공방의 핵심)

1. 개발 단계에서 이슈가 2개 이상 남은 상태로 만든다.
2. 이슈 1개만 처리한 뒤 **세션을 완전히 종료**한다.
3. 새 세션을 열고 `/다음단계 테스트` 입력.
4. **기대**: 공방장이 `이슈등록내역.md`/열린 이슈를 읽고 **남은 이슈부터 이어서** 개발한다.
   - 처음부터 다시 시작하거나, 닫힌 이슈를 또 작업하면 결함.

---

## 4. 자주 나오는 첫 실패 (워치포인트)

- **`gh issue create` 실패**: `projects/테스트/`는 로컬 git repo가 아니므로, repo-manager는
  `gh issue create -R LeeSiHyeon711/테스트 ...` 처럼 **`-R` 플래그로 대상 repo를 명시**해야 한다.
  이게 안 되면 4단계에서 막힌다 → repo-manager 규칙/에이전트를 보강한다.
- **repo 이름 충돌**: 같은 이름 repo가 이미 있으면 생성 실패. 테스트는 매번 새 이름 권장.

---

## 5. 정리 (테스트 후)

- 로컬: `rm -rf projects/테스트`
- GitHub: 테스트 repo는 `gh repo delete`가 settings에서 차단돼 있으므로 **github.com에서 수동 삭제**
  (또는 검증 동안만 임시로 deny 규칙을 완화).
