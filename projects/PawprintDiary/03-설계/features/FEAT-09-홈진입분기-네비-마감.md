# FEAT-09 — 홈 진입 분기 + 네비 연결 + 마감

- 매칭 이슈: #09
- 작성일: 2026-06-22
- 상위 설계서: `03-설계/설계서.md`

## 1. 목적
첫 진입 분기(프로필 유무)와 전역 네비게이션을 실제 화면들과 연결하고, 온보딩/빈 상태와 반응형·시각 완성도를 마감해 "실제 서비스처럼 보이는" 수준으로 끌어올린다. AC-15·16.

## 의존성 (보정 지시 1·8)
- **usePet 시그니처는 FEAT-03 이후의 `{ pet, loading, refresh }` 기준**으로 사용한다.
- **1마리 MVP**: `activePet` 하나 기준으로 진입 분기·네비·가드를 일관 적용. 반려동물 전환/선택/필터 UI 없음.

## 2. 범위
### 구현할 것
- `app/page.tsx`: 진입 시 `petRepo.getActive()` 확인 → 없으면 `/profile/new`, 있으면 `/today`로 라우팅(클라이언트 컴포넌트, 로딩 스플래시 짧게).
- `BottomNav`(FEAT-01) 최종 연결 점검: 4탭 경로/활성표시가 실제 화면과 맞물리는지.
- 첫 사용 온보딩 카피/빈 상태 일관화(프로필 없음/기록 없음/대화 없음).
- 반응형(모바일 우선, 데스크톱 중앙 정렬 카드 폭 제한), 폰트·여백·색 토큰 일관성 최종 점검.
- `빌드노트.md` 마무리(실행·환경변수·Mock 동작·배포 메모).
### 구현하지 않을 것
- 새 기능 추가(2차 확장 항목 일절 금지).
- 각 화면의 핵심 로직 변경(연결·마감만).

## 3. 입력 / 출력
### 입력
- 완성된 FEAT-01~08 화면들, activePet 상태.
### 출력
- 진입 분기 동작, 일관된 네비/온보딩/반응형, 마감된 UI.

## 4. 동작 흐름
1. `/` 진입 → 스플래시(로고+🐾) → `getActive()` → 분기 라우팅(`router.replace`).
2. 각 탭 이동 시 activePet 없으면 공통 가드(프로필 유도) 일관 적용.
3. 빈 상태 3종(프로필/기록/대화) 톤 통일.
4. 전 화면 모바일/데스크톱 레이아웃 점검 후 마감.

## 5. 수정 예상 파일
- `05-개발/app/page.tsx`
- `05-개발/components/BottomNav.tsx`(연결 점검·미세 수정)
- `05-개발/app/layout.tsx`(반응형 컨테이너 폭/여백 마감)
- `05-개발/app/globals.css`(토큰·반응형 미세 조정)
- `05-개발/빌드노트.md`

## 7. 예외 처리
- `getActive()` 실패(IndexedDB 오류) 시 안내 + 새로고침 유도(무한 스플래시 금지).
- 라우팅 중복 호출 방지(마운트 1회).

## 8. 완료 조건
- 첫 진입 시 프로필 없으면 등록 화면, 있으면 오늘 화면으로 자동 이동.
- 탭 닫고 재진입해도 프로필·기록·대화 모두 유지(AC-15).
- 전체 화면이 일관된 따뜻한 톤·반응형으로 "실제 서비스처럼" 보임(AC-16).
- `npm run build` **타입/빌드 에러 0**으로 통과(전역 규칙 — 설계서 7장), `빌드노트.md`에 실행/환경변수/Mock 설명 존재.

### ★ 보정 지시 9 — 마감 시 "키 없는 Mock end-to-end 흐름" 점검 체크리스트(필수)
실제 ANTHROPIC_API_KEY 없이(Mock 모드) 아래 전 흐름이 끊김 없이 동작하는지 마감 단계에서 직접 확인한다.
- [ ] 프로필 등록(`/profile/new`) → activePet 설정 → `/today` 이동
- [ ] 오늘의 발자국 작성·저장(`/today`) — 일기 본문 저장 성공
- [ ] Mock AI 요약 4종 표시 + `entries.ai_summary`에 저장
- [ ] 기록 목록·상세(`/entries`, `/entries/[id]`)에서 저장된 요약 재사용 표시(재호출 없음)
- [ ] AI 질문하기(`/ask`)에서 Mock 응답 표시 + `conversations`에 저장
- [ ] 탭 종료 후 재진입 시 프로필·기록·대화 모두 유지(AC-15)

## 9. 테스트 방법
1. IndexedDB 비운 상태로 `/` 접속 → `/profile/new`로 이동 확인.
2. 프로필 등록 후 `/` 재접속 → `/today`로 이동 확인.
3. 탭 종료 후 재실행 → 프로필·기록·대화 유지 확인(AC-15).
4. 모바일/데스크톱 폭에서 레이아웃 깨짐 없는지 육안 확인(AC-16).
5. 키 없이(Mock) 위 보정 지시 9 체크리스트 6항목 전부 통과 확인.

## 10. 금지 사항

- 다중 반려동물 전환/선택/필터 UI 추가 금지(보정 지시 8).
- 죽음·추모 관련 표현/모드 추가 절대 금지.
- 설계서 외 라이브러리 추가 금지.
