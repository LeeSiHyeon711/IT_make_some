## 1. 개요 — 무엇을 만드는가

무지개연결 모드는 **반려동물이 세상을 떠난 뒤, 보호자가 선택적으로 활성화하는 특별 회고 모드**다.
평상시 서비스(일상 AI 일기)의 핵심 기능이 아니며, 죽음·추모를 서비스 전면에 내세우지 않는다.

핵심 UX는 **리포트가 아니라 채팅형 회고(Rainbow Chat)** 다. 보호자가 떠난 아이와 함께한 시간에 대해
직접 질문하고, AI는 **기록 속 사실 + 보호자의 일기·돌봄 흔적**만 조용히 잇는 **안내자**로만 답한다.

> 목적: 슬픔을 없애는 것이 아니라, 함께한 시간을 조용히 되돌아보고 남겨진 발자국을 하나의 이야기처럼
> 이어볼 수 있도록 돕는 것. "함께했던 시간을 다시 걷는 산책" 같은 느낌.

---

## 2. 절대 원칙 (가드레일 — 이 모드의 정체성)

AI는 아래를 **절대 하지 않는다**.

1. 반려동물 말 대리 ("아이가 ~라고 말해요", "고마워하고 있어요")
2. 감정 단정 ("행복했어요", "행복했을 거예요", "용서했어요")
3. 종교·영적 해석 ("하늘에서 보고 있어요", "무지개다리 건너", "기다리고 있어요")
4. 보호자 죄책감 해소형 위로 / 슬픔 재촉 ("이제 괜찮아져야 해요")
5. 의학적 판단·사망 원인 추측

AI가 **지향하는 방향**.

- 기록에 **실재하는 사실**만 인용한다. ("기록을 보면 함께 산책하거나 곁에 머문 순간이 자주 남아 있어요")
- 보호자의 **꾸준한 기록 = 돌봄·사랑의 흔적**을 조용히 비춘다.
- 감정은 보호자의 몫으로 남긴다. ("어떤 결론을 내리기보다 천천히 바라보게 해줘요", "슬픔을 서둘러 정리하지 않아도 괜찮아요")
- 기록이 적으면 "남겨진 기록이 많지는 않지만…"처럼 조심스럽게 응답한다.

## ★ 보정 지시 (구현 시 반드시 지킨다 — 사용자 확정)

### 보정 지시 R1 — `sanitizeRainbow`는 "단어 삭제기"가 아니라 "위험 문장 패턴 차단기"
- **단어 자체**(예: `고마워`, `사망`, `무지개`, `세상을 떠난`)를 무조건 제거하지 않는다.
- 이유: 사용자가 "고마웠던 순간을 정리해줘"라고 물을 수 있고, 첫 진입 안내문에도 "세상을 떠난 반려동물"·"무지개연결" 같은 표현이 필요하다. 서비스 명칭·안내 문구까지 sanitize되면 UX가 깨진다.
- **차단 대상은 위험한 문장 패턴**이다. (아래 표)

| 차단(금지) | 허용(통과) |
|------------|-----------|
| "아이가 고마워하고 있어요" | "고마웠던 순간을 기록 속에서 찾아볼게요." |
| "아이도 행복했을 거예요" | "무지개연결은 함께한 시간을 돌아보는 공간이에요." |
| "하늘에서 보고 있어요" | "세상을 떠난 반려동물과의 추억을 회상하기 위한 모드입니다." |
| "당신을 용서했을 거예요" | "남겨진 발자국 속에 돌보려 한 흔적이 보여요." |
| "사망 원인은 ~일 수 있어요" | "이 기록은 결론을 내리기보다 천천히 바라보게 해줘요." |

→ 즉, **반려동물을 주어로 한 감정/의지/영적 단정**, **사망 원인 추측**, **슬픔 재촉**의 *문장 형태*를 잡는다. 단어 단순 매칭 금지.

### 보정 지시 R2 — UI 고정 문구와 AI 응답 후처리를 분리한다
- 첫 진입 안내·버튼·모드 설명 등 **UI 문구는 사람이 직접 작성한 고정 문구**다. → `sanitizeRainbow` 대상이 **아니다**.
- `sanitizeRainbow`는 오직 **`/api/ai/rainbow`가 생성한 AI 응답**에만 적용한다.
- UI 안내 문구는 기능 성격을 분명히 알려야 하므로 다음 같은 표현을 그대로 **유지 가능**해야 한다:
  - "무지개연결은 반려동물이 세상을 떠난 뒤, 함께했던 시간을 조용히 돌아보기 위한 공간이에요."

### 보정 지시 R3 — Blob 미전송 (기존 규칙 승계)
- AI 요청 payload에 `pet.profile_image`·`entry.photos`(Blob)를 절대 포함하지 않는다.
- 사진은 **메타데이터(존재 여부·개수)만** 텍스트로 전달한다.

## 3. 형태·스택 결정

- **형태/스택 변경 없음.** 기존 Next.js(App Router)+React+TS+Tailwind+IndexedDB(`idb`) 구조를 그대로 확장한다.
- 새 라이브러리 도입 없음. AI는 기존 서버 프록시 패턴(`@anthropic-ai/sdk`, 키 없으면 결정적 Mock) 그대로.
- 근거: 1차 MVP가 이미 채팅(`/ask`)·AI 프록시·가드레일·대화 저장 인프라를 갖추고 있어, 무지개연결은 **신규 발명 없이 확장**으로 가장 빠르게 검증 가능하다.

## 5. 데이터 구조 변경 (최소 — 스키마 버전 유지 1, 마이그레이션 0)

### 5-1. 활성화 플래그 — `meta` 재사용
| 키 | 값 | 의미 |
|----|-----|------|
| `rainbowMode` | `'on'` / 미존재 | 활성화 상태 |
| `rainbowActivatedAt` | ISO 문자열(선택) | 활성화 시각 |

### 5-2. 채팅 저장 — `conversations` 재사용 + 선택적 `mode` 필드
- `AIConversation`에 **선택적** `mode?: 'ask' | 'rainbow'` 추가.
- IndexedDB는 객체 필드 추가에 **버전업/마이그레이션 불필요**(기존 레코드는 `mode` 없음 → `'ask'`로 간주).
- `/ask`는 `mode` 미지정(기존 동작 불변), `/rainbow`는 `mode:'rainbow'`로 저장·조회 분리.
- **첫 회고 카드(intro)·AI 연결 글은 저장하지 않고 매번 생성.** 저장 대상은 보호자가 실제 주고받은 채팅뿐.

### 5-3. `Pet` 타입 — 변경 없음
- 반려동물을 데이터 모델상 "사망"으로 표시하지 않는다(정체성 보호 + 가역성).

## 6. UX 정의

### 6-1. 진입 (프로필 최하단, 이중 확인)
```
[프로필 화면 최하단]  "무지개연결" 섹션 (조용한 톤)
    │ "무지개연결 열기"
    ▼
[1차 안내]  "무지개연결은 반려동물이 세상을 떠난 뒤, 함께했던 시간을
             조용히 돌아보기 위한 공간이에요. 아직 준비되지 않았다면
             지금 열지 않아도 괜찮아요."
        [아직 아니에요] → 복귀     [계속 보기] ▼
[2차 확인 모달]  "이 기능은 세상을 떠난 반려동물과의 추억을 회상하기 위한
                 모드입니다. 무지개연결 모드를 활성화할까요?"
        [취소할게요] → 저장 안 함   [무지개연결 활성화] ▼
        rainbowMode='on' 저장 → /rainbow
```
- 이미 활성화된 보호자는 섹션이 "무지개연결 들어가기 / 닫기"로 바뀌어 안내를 건너뛴다.

### 6-2. `/rainbow` (채팅 중심)
```
① 조용한 헤더: "무지개연결" / "함께 남긴 발자국을 천천히 돌아보는 공간이에요."
② 첫 회고 카드: 최초 진입 시 전체 기록 기반 짧은 회고 1회 자동 생성
   (항목 나열 리포트 금지 → "함께한 시간의 분위기 / 자주 남겨진 순간 / 돌봄 흔적"을 부드러운 산문)
③ 추천 질문 칩: "가장 자주 기록된 행복한 순간은?" / "내가 꾸준히 돌본 흔적을 보여줘"
   / "기억하고 싶은 장면을 정리해줘" / "우리의 시간을 짧은 편지처럼 정리해줘"
④ 채팅 영역(RainbowChat): 질문 입력 → 회고형 답변 → Q&A 카드 누적(mode:'rainbow' 저장)
⑤ 조용한 닫기: "무지개연결 닫기" → rainbowMode off → 일상 모드 복귀
```
- 미활성(`rainbowMode≠'on'`) 상태로 `/rainbow` 직접 접근 → 안내 후 `/profile`로 차단.

## 7. AI 컨텍스트 / 엔드포인트 설계

### 7-1. 엔드포인트 `POST /api/ai/rainbow`
- 입력: `{ pet, intent: 'intro' | 'chat', question?, records, history? }` (Blob 전면 제외 — 보정 지시 R3)
- 출력: `{ answer: string }`
- 키 없으면 결정적 Mock, 호출 실패 시 graceful Mock 폴백 (기존 패턴 승계).
- `intent`로 첫 회고(intro)와 채팅(chat)을 단일 코드 경로에서 분기 → 향후 확장(사진 메타 회고 등) 시 intent 추가만으로 확장.

### 7-2. 컨텍스트 폭 (회고이므로 `/ask`보다 넓게)
- `/ask`는 최근 5개만 보지만, 회고는 **전체 타임라인을 대표하도록 더 많은 기록(최대 20개)** 을 샘플링.
- 기록 수가 20 초과면 시기별로 분산 추출(처음·중간·최근이 고루 포함되도록).
- 각 기록: `date`, `mood_tags`, `diary_text`(요약 길이로 절단), `ai_summary.condition`, 사진 **개수만**.
- 모든 직렬화는 Blob 제외.

### 7-3. 전용 시스템 프롬프트 (`lib/rainbowPrompt.ts`)
- 역할 고정: "세상을 떠난 반려동물과 함께한 시간을 보호자가 조용히 되돌아보도록 돕는 안내자. 반려동물을 대신해 말하지 않는다."
- 2장 절대 원칙(말대리·감정단정·영적·죄책감/재촉·의료/사망원인 금지) + 지향(사실 인용·돌봄 흔적·감정은 보호자 몫·기록 적을 때 조심) 명시.
- intro/chat 각각의 user 프롬프트 빌더 제공.

## 8. 기존 `/ask`와의 차이 (혼동 방지)

| 항목 | `/ask` | `/rainbow` |
|------|--------|-----------|
| 전제 | 함께 지내는 중 | 세상을 떠난 뒤 회고 |
| 컨텍스트 | 최근 5개 | 전체 샘플(최대 20) + AI요약 + 사진 메타 |
| 시스템 프롬프트 | `SYSTEM_PROMPT` | `rainbowPrompt`(더 엄격) |
| 후처리 | `sanitize` | `sanitizeRainbow`(영적/회고 패턴 추가) |
| vet_note | 조건부 | 없음(의료 전면 배제) |
| 저장 | `mode` 미지정 | `mode:'rainbow'` |
| 진입 | 하단 탭 상시 | 프로필 하단, 이중 확인 후 |

## 10. 인수 기준 (AC — 무지개연결)

- **AC-R01** 프로필 최하단에만 진입점 노출, `BottomNav`에는 없음.
- **AC-R02** 1차 "아직 아니에요" / 2차 "취소"에서 `rainbowMode` 저장 안 됨. 2차 "활성화"까지 통과해야만 `'on'` 저장.
- **AC-R03** 미활성 상태로 `/rainbow` 직접 URL 접근 시 안내 후 차단.
- **AC-R04** `/rainbow` 최초 진입 시 첫 회고 카드 자동 생성(리포트식 항목 나열 아님).
- **AC-R05** 채팅 질문 → 회고형 답변, `mode:'rainbow'`로 저장, 재진입 시 이어지고 `/ask`와 안 섞임.
- **AC-R06** 기록이 거의 없을 때 "남겨진 기록이 많지는 않지만…" 톤으로 응답.
- **AC-R07** AI 응답에 말대리·감정단정·영적·죄책감/재촉·의료/사망원인 **문장**이 출현하지 않음(보정 지시 R1 — 단어 단순 삭제 아님).
- **AC-R08** UI 고정 안내 문구("세상을 떠난 반려동물…", "무지개연결…")는 sanitize되지 않고 그대로 유지(보정 지시 R2).
- **AC-R09** 키 없을 때 Mock 동일 입력 → 동일 출력(결정성).
- **AC-R10** "무지개연결 닫기" → `rainbowMode` off → 일상 모드 정상, 일기/프로필/기존 4탭/`/ask` 무영향.
- **AC-R11** AI 요청 payload에 Blob 미포함(보정 지시 R3).
- **AC-R12** `npm run build` 타입/빌드 에러 0.
