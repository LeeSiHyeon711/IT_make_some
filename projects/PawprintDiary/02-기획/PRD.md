## 1. 제품 개요 및 목표

**PawprintDiary(발자국일기)** 는 강아지·고양이 보호자가 반려동물의 하루를 짧게 기록하면, AI가 행동 해석·컨디션 요약·관찰 포인트·추억 문장을 따뜻하게 정리해주는 AI 반려동물 일기 서비스다.

**MVP 목표**: "프로필 등록 → 오늘의 발자국 작성 → AI 요약/행동 해석 → 기록 확인" 핵심 흐름을 빠르게 구현하고 검증한다. 인증 없이 브라우저 로컬 저장 기반으로 동작하며, 1마리 등록에 집중한다.

**목표 사용자**: 반려동물(강아지·고양이)을 키우는 보호자. 매일 긴 시간을 쓰기 어렵지만 짧은 기록은 꾸준히 남기고 싶고, 특이 행동이나 컨디션 변화의 의미를 전문 지식 없이 파악하고 싶은 사람.

**핵심 문제 해결**:
- 반려동물 일상을 체계적으로 남길 쉬운 도구 제공
- 행동·컨디션 변화의 의미를 AI가 따뜻하고 신중하게 해석
- 함께한 소중한 순간이 기억 속에서 사라지지 않도록 기록으로 보존

## 4. MVP 스코프 (이번 사이클)

### 포함 (P0 — 이번에 반드시 만드는 것)

1. **반려동물 프로필 등록**: 1마리 한정. 로컬 저장. 프로필 이미지 업로드 포함(저장 방식은 3단계 확정).
2. **오늘의 발자국 작성**: 날짜·자유 일기·구조화 상태 항목(식욕·활동량·수면·배변)·특이 행동·기분 태그·사진(선택). 5분 내 입력·저장.
3. **AI 발자국 읽기**: 저장 후 자동 생성 — 컨디션 요약, 행동 해석, 관찰 포인트, 추억 문장. AI 가드레일 전면 적용.
4. **AI에게 질문하기**: 자유 질문 → 프로필+최근 N개 기록/요약 컨텍스트 → AI 응답. 가드레일 동일 적용.
5. **기록 목록 및 상세 보기**: 날짜별 목록 + 상세(입력 내용+AI 요약) 조회.

### 제외 (나중 — 이번 MVP에서 빼는 것)

- 회원 인증(로그인·회원가입·다기기 동기화) — 2차 확장
- 다중 반려동물 등록·전환 UI — 2차 확장. **단, 데이터 구조에는 `pet_id` 필드를 유지해 확장을 준비한다.**
- 반려동물별 기록 필터링 UI — 1마리 MVP이므로 불필요, 2차 확장 시 추가
- 주간 리포트·컨디션 변화 감지
- 사진첩·추억 문장 모아보기
- 이별 후 회고 모드·추모글·편지 생성
- 결제·커뮤니티·SNS 공유·병원 예약
- 모바일 앱 패키징

## 5. 핵심 사용자 흐름 (시나리오)

### 시나리오 1 — 첫 사용 (프로필 등록 후 첫 기록)

1. 사용자가 서비스에 처음 진입한다.
2. 반려동물 프로필 등록 화면이 나타난다(기존 프로필 없을 때 진입점).
3. 이름·종·품종·나이·성별·성격·좋아하는 것·싫어하는 것·건강 특이사항을 입력하고, 프로필 이미지를 선택적으로 업로드한다.
4. 저장하면 로컬에 pet_id가 생성되고 프로필이 보존된다.
5. "오늘의 발자국 작성" 화면으로 이동한다.
6. 날짜(기본값: 오늘)를 확인하고, 식욕·활동량·수면·배변 상태를 단계형 선택지로 고른다.
7. 특이 행동·자유 일기를 텍스트로 입력하고, 기분 태그를 선택한다. 사진은 선택적으로 첨부한다.
8. 저장 버튼을 누르면 기록이 로컬에 저장된다.
9. 즉시 AI 발자국 읽기가 생성되어 — 컨디션 요약·행동 해석·관찰 포인트·추억 문장 — 화면에 표시된다.

### 시나리오 2 — 일상 기록 및 AI 질문

1. 사용자가 서비스에 진입한다. 기존 프로필과 과거 기록이 로컬에서 자동 로드된다.
2. "오늘의 발자국 작성" 화면에서 오늘 상태를 입력하고 저장한다.
3. AI 발자국 읽기를 확인한다.
4. "AI에게 질문하기"를 선택해 자유로운 질문을 입력한다. (예: "요즘 자꾸 문 앞에서 긁는데 왜 그러는 걸까요?")
5. AI가 해당 반려동물 프로필과 최근 N개 기록/요약을 컨텍스트로 참조해 단정 회피·따뜻한 어조로 응답한다.

### 시나리오 3 — 과거 기록 조회

1. 사용자가 기록 목록 화면에 진입한다.
2. 날짜별 기록 목록이 시간역순으로 나열된다.
3. 특정 날짜 기록을 선택하면 상세 화면이 열린다.
4. 상세 화면에는 당일 입력 내용(상태 항목·자유 일기·사진)과 AI가 생성한 요약(컨디션 요약·행동 해석·관찰 포인트·추억 문장)이 함께 표시된다.

## 6. 기능 명세 — MVP 5대 기능 (화면·동작 단위)

### 6-1. 반려동물 프로필 등록

| 항목 | 세부 동작 |
|------|----------|
| 이름 | 텍스트 입력 (필수) |
| 종 선택 | 강아지 / 고양이 / 기타 중 선택 (필수) |
| 품종 | 텍스트 입력 (선택) |
| 나이 | 숫자 또는 텍스트 입력 (필수) |
| 성별 | 선택지 (수컷 / 암컷 / 중성화 여부 포함 가능) (필수) |
| 성격 | 자유 텍스트 입력 (선택) |
| 좋아하는 것 | 자유 텍스트 입력 (선택) |
| 싫어하는 것 | 자유 텍스트 입력 (선택) |
| 건강 특이사항 | 자유 텍스트 입력 (선택) |
| 프로필 이미지 | 파일 선택 후 업로드 (선택). 저장 방식(Base64·IndexedDB 등)은 3단계 확정. |
| 저장 | 필수 항목 입력 완료 시 저장 활성화. 저장 후 로컬에 pet_id 생성·보존. |
| 수정 | 등록 후 언제든 프로필 수정 가능. |

### 6-2. 오늘의 발자국 작성 (일기 입력)

| 항목 | 세부 동작 |
|------|----------|
| 날짜 | 날짜 선택기 (기본값: 오늘). 과거 날짜도 선택 가능. |
| 식욕 상태 | 단계형 선택 — 잘 먹음 / 보통 / 적게 먹음 / 거의 안 먹음 (선택) |
| 활동량 | 단계형 선택 — 매우 활발함 / 보통 / 조용함 / 거의 움직이지 않음 (선택) |
| 수면 상태 | 단계형 선택 — 평소보다 많이 잠 / 보통 / 평소보다 적게 잠 (선택) |
| 배변 상태 | 단계형 선택 — 정상 / 묽음 / 굳음 / 없음 / 이상 있음 (선택) |
| 특이 행동 | 자유 텍스트 입력 (선택) |
| 자유 일기 | 자유 텍스트 입력 (선택) |
| 기분 태그 | 복수 선택 — 활발함 / 차분함 / 예민함 / 피곤해 보임 / 애교 많음 / 식욕 없어 보임 등 (선택) |
| 사진 첨부 | 파일 선택 (선택). 저장 방식은 3단계 확정. |
| 저장 | 날짜 외 모든 항목이 선택 사항이므로 날짜만 있어도 저장 가능. 목표: 5분 이내 완료. |
| 편집·삭제 | 저장 후 해당 기록 수정·삭제 가능. |

### 6-3. AI 발자국 읽기 (AI 생성 요약)

기록 저장 후 자동으로 AI가 4가지 출력을 생성한다.

| 출력 항목 | 설명 |
|-----------|------|
| 오늘의 컨디션 요약 | 입력된 상태 항목을 종합해 오늘 전반적인 컨디션을 문장으로 요약. |
| 행동 해석 | 특이 행동·기분 태그·자유 일기를 바탕으로 가능한 의미를 가능성 중심으로 해석. |
| 관찰 포인트 제안 | 내일 또는 향후 주의 깊게 살펴볼 사항을 구체적으로 제안. |
| 오늘의 한 줄 추억 문장 | 오늘 하루를 따뜻하게 담은 한 줄 문장 생성. |

**AI 가드레일 (5-3 원칙 전면 적용 — 모든 AI 출력 공통)**:

| 규칙 | 기준 |
|------|------|
| 단정 회피 표현 | "기록을 보면", "가능성이 있어요", "단정할 수는 없지만" 등 완충 표현을 반드시 사용. "확실히", "분명히", "100%" 등 단정 표현 금지. |
| 반려동물 말 대리 금지 | "이 아이가 이렇게 말하고 있어요", "지금 행복하대요" 등 반려동물 시점의 말 전달 방식 사용 불가. |
| 단정적 위로 금지 | "100% 행복했어요", "당신을 용서했어요" 등 보호자에게 과도한 확신을 주는 감정적 단정 표현 금지. |
| 의료 진단 금지 | 어떤 경우에도 진단·처방 형태의 응답 생성 불가. 식욕 저하·활동량 감소·배변 이상·통증 의심 행동이 반복 입력될 경우 "수의사 상담을 권장드려요" 문구를 자연스럽게 포함. |
| 보호자 죄책감 자극 금지 | "더 잘 돌봐야 했어요", "당신 때문에 힘들어해요" 등 보호자를 자책하게 만드는 표현 금지. 돌봄의 흔적을 따뜻하게 정리하는 어조 유지. |
| 종별 응답 범위 | 강아지·고양이: 종·품종 특성을 반영한 행동 해석 우선 적용. 기타 종: 일반 관찰 중심 답변으로 제한, 종별 특화 해석 시도 금지. |

### 6-4. AI에게 질문하기

| 항목 | 세부 동작 |
|------|----------|
| 질문 입력 | 보호자가 반려동물 관련 자유 질문을 텍스트로 입력. |
| AI 컨텍스트 | 해당 반려동물 프로필(이름·종·품종·나이·성별·성격·좋아하는 것·싫어하는 것·건강 특이사항) + 최근 N개 기록/요약을 컨텍스트로 포함. N은 3단계 설계 확정. |
| AI 응답 | 위 컨텍스트를 참조해 맞춤 응답 생성. AI 가드레일 동일하게 적용. |
| 대화 보존 | 질문-응답 쌍을 저장해 같은 날 또는 이후에 다시 볼 수 있도록 함. |

### 6-5. 기록 목록 및 상세 보기

| 항목 | 세부 동작 |
|------|----------|
| 목록 화면 | 날짜별 기록을 시간역순으로 나열. 각 항목에 날짜·대표 기분 태그·요약 한 줄 등 미리보기 표시. |
| 상세 화면 | 선택한 날짜의 입력 내용(상태 항목·자유 일기·특이 행동·기분 태그·사진) + AI 생성 요약(4개 항목) 함께 표시. |
| 필터링 | MVP는 1마리이므로 반려동물별 필터링 UI 제외. 날짜 범위 탐색만 지원. |
| 없는 날 표시 | 기록이 없는 날은 "아직 발자국이 없어요" 등 빈 상태 안내. |

## 7. AI 기능 명세 및 가드레일 요약

> 6-3·6-4의 AI 가드레일이 모든 AI 출력에 공통 적용된다. 여기서는 AI 기능 전체 범위와 제한을 정리한다.

### AI 기능 범위 (MVP)

- 오늘 기록 기반 4종 자동 요약 (컨디션 요약·행동 해석·관찰 포인트·추억 문장)

- 주간·월간 패턴 분석 및 리포트

## 11. 수용 기준 (Acceptance Criteria)

MVP가 완성되었다고 볼 수 있는 조건이다. QA 단계에서 이 기준을 기준으로 검증한다.

| AC # | 기능 | 수용 기준 |
|------|------|----------|
| AC-01 | 프로필 등록 | 필수 항목(이름·종·나이·성별) 입력 후 저장 시 로컬에 pet_id가 생성되고 앱 재진입 후에도 유지된다. |
| AC-02 | 프로필 등록 | 프로필 이미지를 업로드하면 저장 후 화면에 표시된다. (저장 방식 무관, 표시 가능 여부로 판단) |
| AC-03 | 오늘의 발자국 작성 | 날짜만 확인하고 다른 항목을 입력하지 않아도 저장이 가능하다. |
| AC-04 | 오늘의 발자국 작성 | 모든 항목(상태·태그·자유 일기·특이 행동·사진)을 입력하고 저장하는 데 5분을 넘지 않는 입력 구조를 갖춘다. |
| AC-05 | 오늘의 발자국 작성 | 저장한 기록은 브라우저 새로고침 후에도 사라지지 않는다. |
| AC-06 | AI 발자국 읽기 | 기록 저장 후 AI가 컨디션 요약·행동 해석·관찰 포인트·추억 문장 4개를 모두 생성해 화면에 표시한다. |
| AC-07 | AI 어조 검증 | AI 출력에 "확실히", "100%", "당신을 용서" 등 단정 표현이 포함되지 않는다. |
| AC-08 | AI 어조 검증 | AI 출력에 "이 아이가 이렇게 말해요" 등 반려동물 말 대리 표현이 포함되지 않는다. |
| AC-09 | AI 어조 검증 | AI 출력에 진단·처방 형태의 의료 표현이 포함되지 않는다. |
| AC-10 | AI 어조 검증 | 식욕 저하·활동량 감소·배변 이상이 입력되면 AI 출력에 병원 상담 권장 문구가 자연스럽게 포함된다. |
| AC-11 | AI에게 질문하기 | 질문 입력 후 AI가 해당 반려동물 프로필+최근 기록을 참조한 맞춤 응답을 생성한다. |
| AC-12 | AI에게 질문하기 | 질문-응답이 저장되어 이후 다시 볼 수 있다. |
| AC-13 | 기록 목록 | 날짜별 기록 목록이 표시되고, 특정 날짜 기록을 선택해 상세 화면으로 진입할 수 있다. |
| AC-14 | 기록 상세 | 상세 화면에 입력 내용과 AI 요약이 함께 표시된다. |
| AC-15 | 로컬 저장 | 브라우저 탭을 닫고 재진입해도 프로필·기록 데이터가 유지된다. |
| AC-16 | UI 완성도 | 실제 서비스처럼 보이는 수준의 UI. 와이어프레임·뼈대 수준이 아닌 디자인 완성도. |

## 12. 설계 단계로 넘기는 결정 사항

기획에서 확정하지 않고 3단계(설계)에서 결정해야 할 사항을 명시한다.

| 항목 | 참고 사항 |
|------|----------|
| 서비스 형태 결정 | 웹(SPA·SSR 등) / 자동화 프로세스 / 기타 형태 중 선택. 포트폴리오 목적과 로컬 저장 기반을 고려. |
| 기술 스택 결정 | 프론트엔드·백엔드(있다면)·AI API 등. |
| 로컬 데이터 저장 방식 | LocalStorage / IndexedDB / OPFS 등 구체적 방식 선택. |
| 사진 저장 방식 | Base64 인라인 저장 vs IndexedDB Binary vs 외부 스토리지. 구현 난이도에 따라 프로필 이미지만 우선 지원하는 방식도 가능. |
| AI API 선택 및 호출 방식 | AI 서비스 제공자(OpenAI·Claude·기타), 프롬프트 설계, 로컬 저장 MVP에서 API 키 관리 방식. |
| AI 질문하기 컨텍스트 N값 | 최근 N개 기록/요약 참조 시 N 결정 (토큰·응답 품질 트레이드오프 고려). |
| AI 요약 저장 방식 | 기록 저장 시 AI 요약 결과를 함께 저장할지, 조회 시 재생성할지. |
| 화면 구조 및 라우팅 설계 | 페이지 구성·라우팅 방식 (기획에서 화면 설계는 다루지 않으므로 3단계 담당). |
