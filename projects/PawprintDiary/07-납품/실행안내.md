# 실행 안내 — 발자국일기 (PawprintDiary)

> 이 안내서는 개발 경험이 없는 분도 따라 하실 수 있도록 썼습니다.
> 순서대로 진행하시면 브라우저에서 발자국일기를 직접 사용해 보실 수 있습니다.

---

## 준비물 확인 — Node.js 설치 여부

발자국일기를 실행하려면 **Node.js**라는 프로그램이 컴퓨터에 설치되어 있어야 합니다.
설치 여부를 먼저 확인해 주세요.

1. 키보드에서 **터미널(Terminal)** 을 엽니다.
   - Mac: Spotlight(`Command + Space`) → "터미널" 검색 → 실행
   - Windows: 시작 메뉴 → "cmd" 검색 → 명령 프롬프트 실행
2. 아래 명령어를 입력하고 Enter를 누릅니다.
   ```
   node -v
   ```
3. `v18.0.0` 또는 그 이상의 숫자가 나오면 설치된 상태입니다. (예: `v20.11.0`)
4. **"command not found"** 라고 나오면 Node.js가 없는 것입니다.
   - [https://nodejs.org](https://nodejs.org) 에 접속해 **"LTS" 버전**을 다운로드해 설치해 주세요.
   - 설치 후 터미널을 껐다가 다시 열고 위 확인을 반복하세요.

---

## 1단계. 코드 폴더로 이동

터미널에서 발자국일기 코드가 있는 폴더로 이동합니다.

```
cd /Users/lsh/Desktop/IT_make_some/projects/PawprintDiary/05-개발
```

> **경로가 다를 경우**: 파인더(Mac) 또는 탐색기(Windows)에서 `IT_make_some > projects > PawprintDiary > 05-개발` 폴더를 열고, 해당 폴더를 터미널 창에 드래그하면 경로가 자동으로 입력됩니다.

이동이 완료되면 터미널 커서 앞에 `05-개발` 이라는 폴더 이름이 보입니다.

---

## 2단계. 필요한 구성 요소 설치

처음 한 번만 실행하는 설치 단계입니다.

```
npm install
```

인터넷에서 필요한 파일을 내려받습니다. 수십 초에서 1~2분 정도 걸릴 수 있습니다.
"added 수백 packages" 같은 메시지가 나오면 완료된 것입니다.

---

## 3단계. 발자국일기 서버 실행

```
npm run dev
```

아래처럼 나오면 정상 실행된 것입니다.

```
  ▲ Next.js 14
  - Local:        http://localhost:3000
```

터미널 창을 닫으면 서버가 꺼지므로, **실행 중에는 터미널을 그대로 유지**해 주세요.

---

## 4단계. 브라우저에서 접속

브라우저(Chrome, Safari, Edge 등)를 열고 주소창에 입력합니다.

```
http://localhost:3000
```

발자국일기 화면이 나타납니다. 처음 접속하면 반려동물 프로필 등록 화면으로 이동합니다.

---

## (선택) AI 기능을 실제로 사용하고 싶을 때

발자국일기는 **AI 키 없이도 전 기능이 동작**합니다. AI 키가 없으면 미리 준비된 따뜻한 예시 응답으로 모든 화면을 체험할 수 있습니다.

**실제 Claude AI(Anthropic)의 응답을 받으시려면:**

1. Anthropic API 키를 발급받으세요. (https://console.anthropic.com)
2. `05-개발` 폴더 안에서 `.env.example` 파일을 복사해 `.env.local` 이라는 이름으로 만듭니다.
   - Mac 터미널 기준:
     ```
     cp .env.example .env.local
     ```
3. `.env.local` 파일을 텍스트 편집기(메모장, VS Code 등)로 열어 아래 부분을 찾습니다.
   ```
   ANTHROPIC_API_KEY=
   ```
4. 등호(`=`) 뒤에 발급받은 키를 붙여넣습니다. 예시:
   ```
   ANTHROPIC_API_KEY=sk-ant-api03-xxxxx...
   ```
5. 파일을 저장한 뒤 서버를 껐다가 다시 켭니다(`npm run dev`).

> **주의**: `.env.local` 파일은 인터넷에 올라가지 않도록 보호되어 있습니다. 키를 다른 사람과 공유하지 마세요.

## 자주 묻는 문제

### "포트 3000이 이미 사용 중"이라는 메시지가 나올 때

다른 프로그램이 이미 3000번 포트를 쓰고 있는 경우입니다.
Next.js가 자동으로 다른 포트(예: 3001)로 실행합니다.
터미널에 표시된 주소(`http://localhost:3001` 등)를 그대로 복사해 브라우저에 붙여넣으면 됩니다.

### 화면이 나타나지 않거나 오류 화면이 보일 때

1. 터미널이 실행 중인지 확인하세요. (`npm run dev` 가 켜져 있어야 합니다.)
2. 브라우저 주소창에 `http://localhost:3000`을 정확히 입력했는지 확인하세요.
3. 문제가 지속되면 터미널에서 `Ctrl + C`로 서버를 끄고 다시 `npm run dev`를 입력하세요.

### 기록한 데이터가 사라졌을 때

발자국일기의 데이터는 **현재 사용하는 브라우저에만 저장**됩니다.
같은 컴퓨터라도 다른 브라우저(예: Chrome → Safari)로 접속하면 기록이 보이지 않습니다.
이 기기, 이 브라우저에서 기록한 내용만 유지됩니다. (MVP 기본 동작)

### 서버를 종료하고 싶을 때

터미널에서 `Ctrl + C` 를 누르면 서버가 꺼집니다.
다시 쓰려면 같은 폴더에서 `npm run dev`를 입력하면 됩니다.
