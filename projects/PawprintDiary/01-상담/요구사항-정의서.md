# 요구사항 정의서 — PawprintDiary (발자국일기)

- 작성일: 2026-06-22
- 고객/요청자: 발자국일기 서비스 기획자 (GitHub: pawprint-diary)

---

## 1. 무엇을 만드는가

반려동물(강아지·고양이)의 하루를 보호자가 짧게 기록하면, AI가 행동 해석·컨디션 요약·관찰 포인트·추억 문장을 따뜻하게 정리해주는 AI 반려동물 일기 서비스.

---

## 2. 누가 쓰는가 (대상 사용자)

**주 사용자**: 반려동물(강아지·고양이)을 키우는 보호자.

**사용 상황 및 사용자 특성**:
- "오늘 이 행동이 무슨 의미일까?" "컨디션이 평소와 다른 것 같아" 같은 질문을 자주 한다.
- "잘 돌보고 있는 걸까?" "나중에 이 순간을 기억하고 싶다"는 감정을 갖고 있다.
- 전문 지식 없이도 반려동물의 상태를 더 잘 이해하고 싶어 한다.
- 매일 긴 시간을 쓰기 어렵지만, 짧은 기록은 꾸준히 남기고 싶다.

---

## 3. 왜 필요한가 (문제 / 목적)

1. **기록 수단 부재**: 반려동물의 일상(식욕·수면·배변·행동)을 체계적으로 남길 쉬운 도구가 없다.
2. **행동 해석의 어려움**: 특이 행동이나 컨디션 변화가 무엇을 의미하는지 혼자 판단하기 어렵다.
3. **기억의 소멸**: 함께한 소중한 순간과 작은 변화가 시간이 지나면 잊힌다.
4. **죄책감 없는 돌봄 기록**: 바쁜 일상 속에서도 "돌봄의 흔적"을 따뜻하게 남기고 싶다.
5. **서비스 포지셔닝**: 추모·이별 서비스를 전면에 내세우지 않고, 평소의 일상 기록으로 자연스럽게 시작한다. 기록이 쌓이면 함께한 시간을 돌아보는 자료가 된다.

---

## 4. 성공 기준

1. **기록 완료**: 보호자가 오늘의 기록(식욕·활동량·수면·배변·특이 행동·자유 일기)을 5분 이내에 입력하고 저장할 수 있다.
2. **AI 읽기 제공**: 기록 저장 후 AI가 오늘의 요약·행동 해석·관찰 포인트·추억 문장을 생성해 보여준다.
3. **기록 조회**: 날짜별·반려동물별로 과거 기록과 AI 요약을 다시 볼 수 있다.
4. **AI 어조 원칙 준수**: "기록을 보면", "가능성이 있어요"처럼 단정을 피하고, 의료 진단·반려동물 말 대리·단정적 위로 없이 따뜻하게 응답한다.

## 5. 핵심 요구사항 (해야 하는 것)

### 5-1. 반려동물 프로필 등록
- [ ] 이름 입력
- [ ] 종 선택 (강아지 / 고양이 / 기타)
- [ ] 품종 입력
- [ ] 나이 입력
- [ ] 성별 선택
- [ ] 성격 입력 (자유 텍스트)
- [ ] 좋아하는 것 입력
- [ ] 싫어하는 것 입력
- [ ] 건강 특이사항 입력
- [ ] 프로필 이미지 업로드

### 5-2. 오늘의 발자국 작성 (일기 입력)
- [ ] 날짜 지정 (기본값: 오늘)
- [ ] 자유 일기 텍스트 입력
- [ ] 식욕 상태 선택 (단계형: 예. 잘 먹음 / 보통 / 적게 먹음 / 거의 안 먹음)
- [ ] 활동량 선택 (단계형)
- [ ] 수면 상태 선택 (단계형)
- [ ] 배변 상태 선택 (단계형)
- [ ] 특이 행동 입력 (자유 텍스트)
- [ ] 오늘의 기분 태그 선택 (예: 활발함 / 차분함 / 예민함 / 피곤해 보임 등)
- [ ] 사진 업로드 (선택 항목)

### 5-3. AI 발자국 읽기 (AI 생성 요약)
- [ ] 오늘의 컨디션 요약 생성
- [ ] 행동 해석 생성 (기록된 행동 기반, 가능성 중심 표현)
- [ ] 관찰 포인트 제안 생성
- [ ] 오늘의 한 줄 추억 문장 생성
- [ ] **AI 답변 원칙 준수** (아래 모두 필수):
  - "기록을 보면", "가능성이 있어요", "단정할 수는 없지만" 등 단정 회피 표현 사용
  - 반려동물 말 대리 금지 ("이 아이가 이렇게 말하고 있어요" 형태 사용 불가)
  - "100% 행복했어요", "당신을 용서했어요" 같은 단정적 위로 금지
  - 의료 진단 금지. 식욕 저하·활동량 감소·배변 이상·통증 의심 행동이 반복 입력될 경우 병원 상담 권장 문구 포함
  - 보호자 죄책감 자극 금지. 돌봄의 흔적을 따뜻하게 정리하는 어조 유지

### 5-4. AI에게 질문하기
- [ ] 보호자가 반려동물과 관련된 자유 질문 입력
- [ ] AI가 해당 반려동물의 축적된 기록을 참조해 응답
- [ ] 위 AI 답변 원칙 동일하게 적용

### 5-5. 기록 목록 및 상세 보기
- [ ] 날짜별 기록 목록 조회
- [ ] 반려동물별 기록 필터링
- [ ] 기록 상세 페이지: 입력 내용 + AI 요약 함께 표시

## 6. 제약 / 전제

### MVP 제외 항목 (명시적으로 확정된 제외)
- 결제 기능
- 커뮤니티·SNS 공유
- 병원 예약
- 전문 수의학 진단
- 반려동물 사칭 채팅 (AI가 반려동물인 척 대화하는 방식)
- 이별 후 회고 모드 완전 구현
- 모바일 앱 패키징 (네이티브 앱 배포)

### 2차 확장 후보 (MVP 이후 순위 미정)
주간 리포트, 컨디션 변화 감지, 사진첩, 추억 문장 모아보기, 이별 후 회고 모드, 추모글·편지 생성, 다중 반려동물 관리, 가족 공동 기록.

### 기타 전제
- **포트폴리오 목적 병행**: 실제 서비스처럼 보이는 수준의 UI/UX 필요. 깔끔하고 완성도 있는 외형.
- **디자인 방향**: 귀엽고 따뜻한 반려동물 일기장 분위기. 밝고 부드러운 톤. 강아지·고양이 모두 어울리는 범용 UI. 유아틱하지 않은 깔끔한 스타일. 죽음·추모를 전면에 내세우지 않음.
- **서비스명**: 영문 PawprintDiary / 한국어 발자국일기. GitHub repo명: pawprint-diary.
- **수의학 원칙 절대 준수**: 어떤 경우에도 의료 진단 형태의 응답 생성 불가.

## 7. 확정된 결정 사항 (2026-06-22 고객 확인 완료)

1. **회원 인증(로그인/회원가입): MVP 미포함 — 로컬 저장 기반으로 시작** ✅ 확정
   1차 MVP는 회원가입·로그인 없이 브라우저 로컬 저장 기반으로 진행한다. 이번 MVP의 핵심은 인증이 아니라 "프로필 등록 → 오늘의 발자국 작성 → AI 요약/행동 해석 → 기록 확인" 흐름을 빠르게 검증하는 것이다. 회원 인증은 다기기 동기화·장기 기록 백업·가족 공동 기록이 필요해지는 2차 확장에서 추가한다.

2. **다중 반려동물: MVP는 1마리로 제한** ✅ 확정
   1차 MVP는 한 보호자가 한 마리를 등록·기록하는 흐름에 집중한다. 다중 반려동물 허용 시 반려동물 선택·기록 필터링·AI 컨텍스트 분리로 화면과 로직이 복잡해지므로 MVP에서 제외한다.
   → **단, 추후 확장을 고려해 데이터 구조는 `pet_id` 기준으로 설계할 것** (설계 단계 반영 권장).
