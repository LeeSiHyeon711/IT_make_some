## 3. 검증 중 발견된 관찰 사항

아래는 AC 위반은 아니지만 UX 관점에서 기록해두는 관찰입니다.

### 관찰-01: /today 재진입 시 기존 당일 기록 미표시

- **현상**: /today 탭을 다른 탭에서 다시 클릭하거나 http://localhost:3000 재접속 후 /today 이동 시, 오늘 이미 작성한 기록이 있어도 항상 빈 일기 입력 폼이 표시됨.
- **기대**: 당일 기록이 이미 있을 경우, 최신 기록의 AI 요약을 보여주거나 "오늘 이미 기록이 있어요" 안내가 있으면 더 자연스러운 흐름.
- **영향**: PRD AC 위반 없음. PRD 요구사항은 "오늘의 발자국 작성" 화면을 제공하는 것이며, 당일 중복 기록 허용 여부를 명시하지 않음. 현 구현은 복수 기록을 허용하는 설계이며, 기록 모음(/entries)에서 전체 확인 가능. 기능 결함 아님.
- **심각도**: 낮음 (UX 개선 사항)

### 관찰-02: favicon.ico 미배치

- **현상**: public/favicon.ico 파일 없음 → 브라우저 탭 아이콘 없음.
- **영향**: 기능 무관. 포트폴리오 완성도 측면에서 개선 가능.
- **심각도**: 낮음 (미관)
