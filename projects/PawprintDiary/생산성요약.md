## 개선 메모 (이번 프로젝트 관찰)

- **Bash 차단 이슈**: builder 에이전트가 터미널 명령을 직접 실행할 수 없어 공방장이 `npm run build`, `git commit/push`, `gh` 명령을 대행함. 향후 builder 환경에서 Bash 도구 허용 여부 검토 필요.
- **Mock 전략의 효과**: AI 키 없이 전체 흐름을 검증할 수 있는 Mock 구현(FEAT-05)이 QA 진입 장벽을 낮추는 데 효과적이었음. 실키 없이도 AC-06~14 전항목 검증 가능.
- **FEAT 문서 품질**: 보정 지시 10건이 FEAT 문서에 명시됨으로써 Sonnet 레벨 builder가 Opus 없이도 안정적으로 구현. "FEAT 품질 = 결과 품질" 원칙이 실제로 작동했음.
- **빌드 게이트 효과**: 매 FEAT 완료 시 `npm run build` 에러 0 강제 → 최종 QA에서 빌드 관련 결함 0건.
