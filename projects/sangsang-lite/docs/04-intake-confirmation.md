# 04 · 공방 접수 확인 (Intake Confirmation)

> **공방 접수 확인은 별도 에이전트가 아니다.** 소통 에이전트가 만든 **확인 단계(승인 게이트)**다.
> 사용자 노출명: 🧾 공방 접수 확인

## 1. 역할

- AI가 사용자의 아이디어를 **잘못 이해한 상태로 분석에 들어가는 것을 방지**한다.
- 사용자가 자신의 아이디어를 **한 번 더 정리**하게 한다.
- 이후 **균열점 진단에 대한 반발감을 줄인다** (이미 동의한 내용을 바탕으로 진단하므로).

> 이 단계는 워크플로의 **유일한 사람 승인 게이트**다. (02 문서 참조)
> **승인 후에만** 균열점 진단으로 넘어간다.

---

## 2. 사용자 노출 카드

```
🧾 공방 접수 확인

제가 이해한 아이디어는 아래와 같아요.

만들고 싶은 것:
{아이디어 요약}

해결하고 싶은 문제:
{문제}

처음 쓸 사람:
{예상 사용자}

아이디어 출처:
{직접 경험 / 주변인 관찰 / 시장 추정 / 상상 확장}

현재 아이디어 상태:
{막연함 / 문제 정의됨 / 해결책까지 있음}

검증에 쓸 수 있는 시간:
{사용자가 선택한 시간}

이 내용으로 공방 작업을 시작해도 될까요?
1. 좋아요, 진행해주세요
2. 수정하고 싶어요
```

---

## 3. 내부 필드 → 사용자 표현 매핑

| 내부 필드 | 카드 표시 | 변환 규칙 |
|-----------|-----------|-----------|
| `input_summary` | 만들고 싶은 것 | 그대로 (사용자 언어로) |
| `problem` | 해결하고 싶은 문제 | 그대로 |
| `target_user` | 처음 쓸 사람 | 그대로 |
| `pain_source` | 아이디어 출처 | SELF=직접 경험 / OBSERVED=주변인 관찰 / ASSUMED=시장 추정 / IMAGINED=상상 확장 |
| `maturity` | 현재 아이디어 상태 | RAW=막연함 / SITUATION·PROBLEM=문제 정의됨 / SOLUTION=해결책까지 있음 |
| `validation_time_budget` | 검증에 쓸 수 있는 시간 | 30분 이내 / 오늘 안에 / 2일 이내 / 1주일 이내 / 2주 이상 |

> 카드에는 **JSON 용어(SELF, RAW 등)를 절대 노출하지 않는다.** 사용자 언어로 바꿔 보여준다.

---

## 4. 분기 처리

- **1. 진행** 선택 → 승인된 접수 JSON을 **아이디어 진단 에이전트**(05)로 전달.
- **2. 수정** 선택 → 사용자가 어떤 항목을 고치고 싶은지 듣고, 해당 필드만 갱신한 뒤 **카드를 다시 보여준다.**
  - 수정은 여러 번 가능하지만, 매번 **전체를 다시 묻지 않고** 바뀐 부분만 갱신한다.

---

## 5. 원칙

- 접수 확인 없이 진단으로 넘어가지 않는다.
- 카드는 **짧고 읽기 쉽게** — 사용자가 5초 안에 "맞다/아니다"를 판단할 수 있어야 한다.
- 비어 있는 필드는 "(아직 정해지지 않음)"처럼 부드럽게 표시하고, 진단 단계에서 다룬다.
