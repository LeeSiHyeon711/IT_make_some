# 요구사항 정의서 — PickUpMemo

- 작성일: 2026-06-15
- 고객/요청자: 배달기사 겸 개인 개발자 (1인)

---

## 1. 무엇을 만드는가

배민커넥트 배차 화면에서 표시되는 업체 상호(식당명)를 Android 접근성 API 및 알림 데이터로 추출 가능한지 검증하기 위한 기술 검증 전용 로그 수집기 앱 (PickUpMemo v1)

> 주의: 이번 버전은 "기술 타당성 검증 도구"이며 실제 서비스 앱이 아니다. 메모·팝업 기능은 v2에서 다룬다.

---

## 2. 누가 쓰는가 (대상 사용자)

- 주 사용자: 개발자 본인 (배달기사 개인)
- 사용 상황: 본인 Android 기기에 앱을 설치하고, 실제 배민커넥트 앱 사용 중 로그를 수집한 뒤 파일로 내보내 데이터를 분석한다.
- 사용 규모: 단일 기기, 1인 사용

---

## 3. 왜 필요한가 (문제 / 목적)

향후 개발 예정인 PickUpMemo v2(배차 시 식당 메모 자동 팝업)를 구현하려면, 배민커넥트 앱이 Android 시스템에 노출하는 접근성 트리와 알림 데이터에서 업체 상호를 실제로 읽어올 수 있는지 먼저 확인해야 한다.

- 검증 성공 시: v2 개발 착수 (업체명 기반 메모 자동 팝업)
- 검증 실패 시: OCR 또는 다른 대안 방식 검토

검증 없이 v2를 개발하면 핵심 기능 자체가 불가능할 수 있으므로, v1 기술 검증이 선행 필수 조건이다.

---

## 4. 성공 기준

실제 배민커넥트 앱에서 배차가 발생했을 때, 로그 수집 앱이 내보낸 파일 안에 업체 상호(예: "교촌치킨 죽전점") 또는 이를 유추할 수 있는 데이터가 포함되어 있어야 한다.

검증 시나리오 (순서대로):
1. 앱 설치
2. 접근성 권한 허용
3. 알림 접근 권한 허용
4. 배민커넥트 배차 발생 (실제 배차 또는 데모 화면 활용)
5. 로그 내보내기 실행 (공유 Intent → txt 파일 전송)
6. 내보낸 txt 파일에서 업체 상호 데이터 육안 확인

---

## 5. 핵심 요구사항 (해야 하는 것)

### 5-1. 데이터 수집
- [ ] 접근성 서비스 구현: 화면에 표시되는 텍스트, ContentDescription, View ID, Package Name, Activity 정보를 수집하여 로그에 기록
- [ ] 알림 수신기 서비스 구현: 알림 제목, 알림 내용, 패키지명, 발생 시각을 수집하여 로그에 기록

### 5-2. 로그 저장
- [ ] 수집된 로그를 기기 내부 저장소에 텍스트 형식으로 저장
- [ ] 로그 항목 형식 예시:
  ```
  [2026-06-15 14:33:12]
  Package: com.example
  Type: Accessibility
  Text: 교촌치킨 죽전점
  ```
- [ ] 외부 서버 전송 없음 (기기 내부 저장만)

### 5-3. 메인 화면
- [ ] 접근성 서비스 실행 상태 표시
- [ ] 접근성 권한 허가 상태 표시
- [ ] 알림 접근 권한 허가 상태 표시
- [ ] 로그 보기 화면 이동 버튼
- [ ] 로그 내보내기 버튼

### 5-4. 로그 조회 화면
- [ ] 수집된 로그 목록 표시 (최신순 정렬)
- [ ] 패키지명 기준 필터: 패키지명 입력 시 해당 앱의 로그만 표시
- [ ] 전체 로그 삭제 기능
- [ ] 디자인 수준: 기능 동작 확인 가능한 최소 수준 (완성도 불필요)

### 5-5. 로그 내보내기 (필수)
- [ ] "로그 내보내기" 버튼 실행 시 txt 파일 생성
- [ ] Android 공유 Intent 실행으로 카카오톡, Gmail, Google Drive 등으로 공유 가능

### 5-6. 데모 / 테스트 편의 기능
- [ ] 테스트 알림 생성 버튼: 제목 "신규 배달 요청", 내용 "교촌치킨 죽전점 / 배달료 3,900원" 알림 발생 → 알림 수신기 로그 수집 여부 확인용
- [ ] 접근성 테스트 화면: 아래 텍스트를 포함한 단순 모의 배차 화면 → 접근성 서비스 수집 여부 확인용
  - 표시 텍스트 항목: 업체 상호, 교촌치킨 죽전점, 전달지 주소, 배차 수락, 거절

## 6. 제약 / 전제

### 절대 구현 금지 항목 (고객이 명시한 v1 범위 외)
- 회원가입 / 로그인
- 서버 / 데이터베이스 서버 / 클라우드 저장
- AI 기능 / OCR
- 메모 기능 / 팝업 기능

### 고객이 명시한 기술 환경 (설계 단계에서 최종 확정)
- 플랫폼: Android Native
- 언어: Kotlin
- UI: Material 3
- 최소 지원 버전: Android 8.0 (API 26) 이상

### 최종 납품 요구사항
- Android Studio에서 바로 열 수 있는 전체 프로젝트 소스
- 디버그 APK 빌드 가능 상태
- APK 설치 방법 안내 문서
- 접근성 권한 허용 방법 안내 문서
- 알림 접근 권한 허용 방법 안내 문서
- 테스트 절차 문서

### 개발 우선순위 (고객이 명시)
1순위: 로그 수집 (접근성 서비스 + 알림 수신기)
2순위: 로그 내보내기
3순위: 로그 조회 UI
최하위: 디자인

---

## 7. 미확정 사항 (다음 단계로 넘기는 질문)

아래 항목은 고객 요청에 명시되지 않았으므로 기획/설계 단계에서 결정한다.

1. 로그 내보내기 파일명 형식: 자동 생성 규칙을 정해야 한다. (예: `pickupmemo_log_20260615_143312.txt`)
2. 로그 보관 한도: 기기 저장 용량 소모 방지를 위한 최대 보관 건수 또는 기간 제한 필요 여부
3. 필터 조건 영속성: 패키지명 필터 설정을 앱 재시작 후에도 유지할 것인지 여부
