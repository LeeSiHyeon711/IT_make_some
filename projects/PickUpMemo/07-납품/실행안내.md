# PickUpMemo v1 실행안내

> 이 문서는 PickUpMemo 앱을 처음 설치하거나, 나중에 다시 세팅해야 할 때 처음부터 끝까지 따라 할 수 있도록 작성했습니다.

---

## 1. 납품 파일 위치

| 항목 | 경로 |
|------|------|
| 설치 파일 (APK) | `projects/PickUpMemo/05-개발/PickUpMemo/app/build/outputs/apk/debug/app-debug.apk` (약 6.6MB) |
| 소스 코드 프로젝트 | `projects/PickUpMemo/05-개발/PickUpMemo/` 폴더 전체 |

APK 파일 하나만 있으면 바로 설치할 수 있습니다. 소스 코드는 나중에 기능을 추가하거나 직접 다시 빌드할 때 씁니다.

---

## 2. 안드로이드 폰에 앱 설치하기

방법이 두 가지입니다. 더 편한 쪽을 선택하세요.

### 방법 A — USB 케이블 + adb 명령 (추천)

1. 폰을 USB 케이블로 PC에 연결합니다.
2. 폰 화면에 "USB 디버깅을 허용하시겠습니까?" 팝업이 뜨면 **허용**을 누릅니다.  
   (팝업이 안 뜨면: 폰 설정 → 개발자 옵션 → USB 디버깅 켜기)
3. PC 터미널(또는 명령 프롬프트)에서 아래 명령을 실행합니다.

```
adb install "경로/projects/PickUpMemo/05-개발/PickUpMemo/app/build/outputs/apk/debug/app-debug.apk"
```

4. `Success` 메시지가 뜨면 설치 완료. 폰 앱 목록에서 **PickUpMemo** 아이콘을 찾아 실행합니다.

### 방법 B — 파일 직접 복사

1. `app-debug.apk` 파일을 폰으로 복사합니다 (카카오톡 파일 전송, Google Drive 업로드 후 폰에서 다운로드 등).
2. 폰에서 그 파일을 열면 설치 화면이 뜹니다.
3. "출처를 알 수 없는 앱" 경고가 뜰 수 있습니다. **설치 허용**을 누릅니다.  
   (경로는 폰 제조사마다 조금 다를 수 있습니다. 예: 설정 → 보안 → 알 수 없는 출처 허용)

## 3. 접근성 권한 허용하기 (화면 텍스트 수집용)

이 권한이 있어야 배민커넥트 배차 화면의 텍스트를 읽을 수 있습니다.

1. 앱을 실행합니다.
2. 메인 화면에서 **[접근성 설정 열기]** 버튼을 누릅니다.
3. 폰 시스템 설정 화면이 열립니다.
4. 목록에서 **PickUpMemo**를 찾아 누릅니다.
5. 토글을 **켬** 으로 바꿉니다.
6. "이 앱이 모든 화면의 텍스트를 읽을 수 있습니다" 같은 확인 팝업이 뜨면 **허용(확인)**을 누릅니다.
7. 뒤로가기로 PickUpMemo 앱으로 돌아옵니다.
8. 메인 화면 상단에 **"접근성 서비스: 실행 중 / 접근성 권한: 허가"** 로 바뀌었으면 성공입니다.

> 폰 브랜드에 따라 "접근성" 메뉴가 "특수 기능" 또는 "손쉬운 사용" 이름으로 보일 수 있습니다.

---

## 4. 알림 접근 권한 허용하기 (알림 수집용)

이 권한이 있어야 배민커넥트 배차 알림 내용을 읽을 수 있습니다.

1. 메인 화면에서 **[알림 접근 설정 열기]** 버튼을 누릅니다.
2. 폰 시스템 설정 화면이 열립니다.
3. 목록에서 **PickUpMemo**를 찾아 토글을 **켬** 으로 바꿉니다.
4. 뒤로가기로 돌아옵니다.
5. 메인 화면에 **"알림 접근 권한: 허가"** 로 바뀌었으면 성공입니다.

두 권한이 모두 허용된 상태가 완전한 수집 준비 상태입니다.

## 5. 테스트 절차 (처음부터 끝까지)

### 5-1. 데모 테스트 — 앱이 실제로 수집하는지 먼저 확인

**접근성 수집 확인**

1. 메인 화면에서 **[접근성 테스트 화면]** 버튼을 누릅니다.
2. 모의 배차 화면이 열립니다. 화면에 "교촌치킨 죽전점" 같은 텍스트가 보입니다.  
   (버튼은 데모용이라 누르면 안내 메시지만 나옵니다 — 실제 배차 아님)
3. 뒤로가기로 메인 화면으로 돌아와 **[로그 보기]** 를 누릅니다.
4. 방금 화면의 텍스트가 아래처럼 기록되어 있으면 접근성 수집 정상입니다.

```
[2026-06-16 14:33:12]
Package: com.itmakesome.pickupmemo
Type: Accessibility
Text: 교촌치킨 죽전점 | desc=업체상호 | id=tv_store_name
```

**알림 수집 확인**

1. 메인 화면에서 **[테스트 알림 생성]** 버튼을 누릅니다.
2. 안드로이드 13(API 33) 이상 폰이면 "알림을 표시하도록 허용하시겠습니까?" 팝업이 뜹니다. **허용**을 누릅니다.
3. 상태바에 "신규 배달 요청 / 교촌치킨 죽전점 / 배달료 3,900원" 알림이 뜨면 정상입니다.
4. **[로그 보기]** 에서 아래처럼 기록되어 있으면 알림 수집 정상입니다.

```
[2026-06-16 14:33:45]
Package: com.itmakesome.pickupmemo
Type: Notification
Title: 신규 배달 요청
Body: 교촌치킨 죽전점 / 배달료 3,900원
```

> 알림 접근 권한이 꺼진 상태라면 알림이 뜨더라도 로그에는 남지 않습니다. 4장에서 권한을 먼저 허용했는지 확인하세요.

### 5-2. 실제 검증 — 배민커넥트 배차 수신

1. 두 권한이 모두 켜진 상태에서 배민커넥트 앱을 사용하면서 배차를 기다립니다.
2. 배차 화면이 뜨거나 배차 알림이 오면 PickUpMemo가 자동으로 수집합니다. 별도로 할 일은 없습니다.
3. 배차 수신 후 PickUpMemo의 **[로그 보기]** 에서 업체 상호가 기록되었는지 확인합니다.

### 5-3. 로그 내보내기 (PC나 카카오톡으로 파일 보내기)

1. 메인 화면에서 **[로그 내보내기]** 버튼을 누릅니다.
2. 파일 이름이 자동으로 정해집니다. 예: `pickupmemo_log_20260616_143312.txt`
3. 공유 창이 뜹니다. 카카오톡·Gmail·Google Drive 중 원하는 곳으로 보내면 됩니다.
4. 받은 txt 파일을 PC에서 열어 업체 상호 데이터가 들어있는지 확인합니다.

### 5-4. 로그 화면 활용

- **패키지명 필터**: 로그 보기 화면 상단 입력란에 패키지명 일부(예: `baemin` 또는 `com.skt`)를 입력하고 **[적용]** 을 누르면 해당 앱 로그만 볼 수 있습니다. 입력란을 비우고 다시 [적용]하면 전체 로그로 돌아옵니다.
- **전체 삭제**: 로그를 처음부터 다시 쌓고 싶을 때 **[전체 삭제]** 를 누릅니다. 확인 팝업 후 삭제됩니다.
- 앱을 껐다 켜도 필터 설정은 초기화(전체 표시 상태)로 돌아옵니다. 이는 의도된 동작입니다.

## 6. 소스 코드에서 직접 빌드하는 법 (재빌드 필요 시)

Android Studio로 수정·재빌드가 필요할 때만 참고하면 됩니다.

**Android Studio로 프로젝트 열기**

1. Android Studio를 실행합니다.
2. **File → Open** 에서 `projects/PickUpMemo/05-개발/PickUpMemo/` 폴더를 선택합니다.  
   (주의: `05-개발/` 아래 `PickUpMemo/` 폴더를 여는 것입니다. `05-개발/` 폴더 자체가 아닙니다)
3. Gradle Sync가 자동으로 시작됩니다. 인터넷이 연결되어 있으면 필요한 파일을 자동으로 내려받습니다.
4. Sync가 끝나면 **Build → Build Bundle(s)/APK(s) → Build APK(s)** 로 APK를 빌드합니다.

**터미널에서 빌드하는 경우 (JAVA_HOME 주의)**

```bash
cd projects/PickUpMemo/05-개발/PickUpMemo

# ★ 아래 줄이 빠지면 빌드가 실패합니다. 반드시 먼저 실행하세요.
export JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"

./gradlew assembleDebug
```

빌드가 성공하면 `BUILD SUCCESSFUL` 메시지가 나오고, APK는 `app/build/outputs/apk/debug/app-debug.apk` 에 생성됩니다.

> JAVA_HOME 설명: 이 앱의 빌드 도구(AGP 8.5.2)는 JDK 17 이상이 필요합니다. Android Studio에 내장된 JDK(JDK 21)를 쓰도록 위 명령을 먼저 실행해야 합니다. Android Studio 내부에서 Build APK를 사용하면 이 설정이 자동으로 적용됩니다.

## 7. 알려진 한계와 주의 사항

| 항목 | 내용 |
|------|------|
| 권한 허용 방식 | 접근성·알림 접근 권한은 코드로 자동 켤 수 없습니다. 반드시 시스템 설정에서 직접 허용해야 합니다 (Android 보안 정책). |
| 알림 수집 조건 | 알림 접근 권한이 꺼져 있으면 테스트 알림이 뜨더라도 로그에 남지 않습니다. 권한 먼저 확인. |
| 로그 보관 한도 | 최대 5,000건. 초과 시 가장 오래된 것부터 자동으로 지워집니다. 중요한 로그는 쌓이기 전에 내보내기를 권장합니다. |
| 외부 전송 없음 | 수집 데이터는 폰 내부에만 저장됩니다. 인터넷으로 전송되지 않습니다. 로그를 꺼내려면 [로그 내보내기]를 수동으로 눌러야 합니다. |
| 수집 범위 | 접근성 권한을 켜면 배민커넥트뿐 아니라 다른 앱 화면 텍스트도 함께 수집됩니다. 패키지명 필터로 원하는 앱만 볼 수 있습니다. |
| 자동 QA 생략 | 에뮬레이터 환경 제약으로 자동화 테스트를 진행하지 않았습니다. 실기기 수동 테스트로 대체하여 정상 동작을 확인했습니다. |
| 폰 제조사별 차이 | 접근성·알림 설정 메뉴 이름과 위치가 삼성·LG·소니 등 제조사마다 조금씩 다를 수 있습니다. 설정 앱에서 "접근성" 또는 "알림"으로 검색해 찾으시면 됩니다. |
| 디자인 수준 | 이 앱은 기술 검증 목적으로 만들어졌습니다. 화면 디자인은 기능 확인이 가능한 최소 수준으로 만들었습니다. |
