# Operator Profile — AI-Morning-Brief

이 문서는 AI-Morning-Brief가 뉴스를 분석할 때 사용할 운영자 관심도 기준이다.  
목표는 일반적인 AI 뉴스 요약이 아니라, IT상상공방 운영자에게 실질적으로 도움이 되는 AI 동향 브리핑을 생성하는 것이다.

---

## 1. 운영자 관심 분야

운영자는 다음 주제에 높은 관심을 가진다.

- Claude Code
- OpenAI Codex / Codex CLI / Codex GitHub Action
- MCP / PlayMCP / MCP 서버 구축
- AI Agent / Multi-Agent / Agent Orchestration
- GitHub Actions 기반 자동화
- n8n / 워크플로우 자동화
- AI 기반 개발 파이프라인
- 하네스 엔지니어링
- 바이브코딩 / AI Builder / AI Engineer 업무 방식
- 개인용 자동화 도구
- 소규모 MVP 제작 자동화
- IT상상공방에 적용 가능한 개발 프로세스

---

## 2. 특히 중요하게 볼 소식

다음 조건에 해당하는 소식은 높은 우선순위로 판단한다.

1. Claude Code, Codex, Cursor, Gemini CLI 등 AI 코딩 도구의 기능 업데이트
2. GitHub Issue, Pull Request, CI/CD와 AI Agent를 연결하는 사례
3. MCP 서버, MCP 클라이언트, PlayMCP, Tool Calling 관련 업데이트
4. AI Agent를 역할별로 나누어 협업시키는 구조나 사례
5. 개발 자동화, QA 자동화, 문서화 자동화 사례
6. 개인 또는 소규모 팀이 AI로 생산성을 크게 높인 사례
7. 상상공방 v2 아키텍처에 직접 적용 가능한 아이디어
8. 비용이나 토큰 사용량을 줄일 수 있는 운영 전략
9. 로컬 자동화, Discord/Telegram 알림, SQLite 기반 지식 축적 사례
10. AI 개발 도구의 보안 이슈, prompt injection, GitHub Action 권한 문제

---

## 3. 낮은 우선순위 소식

다음 소식은 중요도가 낮다.

- 일반적인 AI 투자 뉴스
- 단순 기업 홍보성 보도자료
- 모델 성능 수치만 강조하고 실사용 흐름이 없는 기사
- 이미지/영상 생성 모델 업데이트 중 상상공방과 직접 관련이 낮은 내용
- 대기업 인수합병 뉴스 중 개발 자동화와 관련 없는 내용
- 너무 학술적이어서 단기 적용이 어려운 논문 소개
- 일반 소비자용 챗봇 기능 업데이트 중 개발 파이프라인과 무관한 내용

---

## 4. 상상공방 적용 가능성 판단 기준

기사가 다음 질문 중 하나 이상에 답할 수 있으면 “상상공방 적용 가능성 있음”으로 판단한다.

- 이 소식을 상상공방 v2 에이전트 구조에 적용할 수 있는가?
- 상담가, 기획자, 설계자, repo-manager, builder, reviewer, triage, delivery 중 특정 에이전트의 성능을 개선할 수 있는가?
- GitHub Issue, FEAT 문서, 자동 QA, Discord 보고, DB 축적 구조와 연결할 수 있는가?
- 토큰 비용을 줄이거나 자동화 안정성을 높일 수 있는가?
- 고객용 MVP 제작 속도나 품질을 높일 수 있는가?
- 포트폴리오에서 “AI 개발 조직 운영체계”를 보여주는 근거가 될 수 있는가?

---

## 5. 중요도 점수 기준

- 5점: 상상공방 구조에 바로 반영하거나 실험할 가치가 있는 소식
- 4점: Claude Code, Codex, MCP, Agent 자동화에 직접 관련된 중요 소식
- 3점: 개발 생산성이나 AI 트렌드 파악에 참고할 만한 소식
- 2점: 업계 동향상 알아두면 좋은 일반 소식
- 1점: 현재 관심사와 관련성이 낮거나 홍보성에 가까운 소식

---

## 6. 리포트 작성 방식

리포트는 단순 요약보다 “운영자에게 어떤 의미가 있는지”를 중심으로 작성한다.

각 중요 소식은 가능하면 다음 형식으로 정리한다.

- 소식 요약
- 왜 중요한가
- 상상공방 적용 가능성
- 오늘 또는 이번 주에 해볼 액션
- 원문 링크

Discord 메시지는 2000자 이내로 짧게 작성한다.  
전체 분석은 Markdown 일일 리포트에 저장한다.

---

## 7. 액션 아이템 기준

액션 아이템은 너무 거창하지 않게 작성한다.  
가능하면 30분~2시간 안에 실험 가능한 단위로 제안한다.

좋은 액션 예시:

- AGENTS.md에 Codex QA 역할 초안 추가 검토
- Claude Code headless 실행 샘플 테스트
- Discord Webhook 전송 실패 처리 체크
- MCP 관련 신규 기능을 상상공방 설계자 에이전트에 반영 가능한지 조사
- GitHub Action 권한 정책을 repo-manager 규칙에 추가 검토

나쁜 액션 예시:

- AI 플랫폼 전체 재설계
- 모든 에이전트 구조 변경
- 새 웹 대시보드 개발
- 당장 상용 서비스 출시
