## 4. 발견 이슈 목록

### ISSUE-01 (경미) — pytest 실행 후 `data/raw/` 빈 디렉토리 잔류

- 심각도: 낮음
- 내용: 테스트 fixture의 `TEST_RAW = "data/raw/test"`로 설정되어 있어, pytest 실행 중 `data/raw/test`를 생성할 때 상위 디렉토리 `data/raw`가 함께 생성된다. `clean` fixture의 teardown이 `data/raw/test`(리프)만 제거하고 `data/raw`는 제거하지 않아 빈 디렉토리가 잔류한다.
- 영향: 실제 데이터 오염 없음 (파일 없는 빈 디렉토리). 운영 DB·리포트·raw 파일은 생성되지 않음. `.gitignore`에 `data/` 포함되어 있어 git에는 추적되지 않음.
- 수정 제안: teardown에서 `shutil.rmtree(os.path.join(ROOT, "data/raw/test"), ...)` 후 `os.path.join(ROOT, "data/raw")`가 비어 있으면 함께 제거하거나, TEST_RAW 경로를 `/tmp/amb_test_raw`와 같은 완전히 독립적인 경로로 변경.

### ISSUE-02 (경미) — `config.yaml`에 `request_timeout_sec` 키 미포함

- 심각도: 낮음
- 내용: `pipeline.py`가 `pipeline_cfg.get("request_timeout_sec", 15)`로 타임아웃을 읽으나, `config.yaml`의 `pipeline` 섹션에 해당 키가 없어 항상 하드코딩 기본값 15초가 사용된다. 설정 파일에서 타임아웃을 조정할 수 없는 상태.
- 영향: 실제 동작 문제 없음 (15초 기본값은 적절). 단, 사용자가 `config.yaml`에서 타임아웃을 조정하려 해도 반영되지 않음.
- 수정 제안: `config.yaml`의 `pipeline` 섹션에 `request_timeout_sec: 15` 항목 추가 및 README 환경설정 섹션에 안내 추가.
