## 이번에 추가된 것

**7일치 AI 뉴스를 종합해 "이번 주 흐름"을 Discord로 전송하는 주간 리포트 기능**입니다.

일간 브리핑이 매일 아침 "오늘 어떤 기사가 중요했나"를 알려준다면, 주간 리포트는 **일주일 단위로 흐름을 정리해 트렌드가 보이는 요약**을 제공합니다. 매주 일요일 새벽 4시 40분(일간 브리핑 10분 뒤)에 자동 실행되도록 설계됐습니다.

---

## 주간 리포트가 분석하는 5가지 층

단순히 기사를 모아 요약하는 것이 아니라, 데이터 기반 신호를 먼저 뽑아낸 뒤 Claude가 흐름을 서술하는 방식입니다.

| 분석 층 | 알 수 있는 것 |
|---------|--------------|
| 일자별 강도 추이 | 이번 주 어느 날 AI 뉴스가 가장 집중됐는지 |
| 태그 빈도 × 지속성 | 일주일 내내 꾸준히 나온 키워드(지속 줄기) vs 하루 반짝 터진 키워드(단발 버스트) 구분 |
| 소스 신호 품질 | AI와 무관한 기사를 많이 뿌린 소스를 자동으로 찾아냄(노이즈 소스 적발) |
| 공방 관련 픽 | 7일치 기사 중 IT상상공방 관련도 7점 이상인 기사만 추려 제시 |
| 상상공방 아이디어 통합 | 일간 브리핑의 "오늘 실험해볼 것" 7일치를 하나로 통합 |

Discord에는 이 5개 층을 9개 블록으로 나눈 다이제스트(2000자 이내)가 전송됩니다. 전문은 `reports/<연도>/weekly/W##.md` 파일로 저장됩니다.

## 일요일 자동 발화 — 직접 켜셔야 합니다

일요일 새벽 자동 실행은 코드만으로는 켜지지 않습니다. **한 번 등록** 절차를 마쳐야 합니다.

절차 요약 (상세는 동봉된 `실행안내-주간보고.md` 참조):

1. `scripts/com.itsangsang.morningbrief.weekly.plist` 파일을 열어 `/ABS/PATH/TO` 세 곳을 실제 경로로 교체
2. `cp scripts/com.itsangsang.morningbrief.weekly.plist ~/Library/LaunchAgents/` 로 복사
3. `launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.itsangsang.morningbrief.weekly.plist` 로 등록

등록 후에는 **매주 일요일 새벽 4시 40분에 Mac이 혼자 주간 리포트를 생성하고 Discord로 보냅니다.** Mac이 절전 중이어도 깨어난 직후 실행합니다. 같은 주에 실수로 두 번 실행되더라도 중복 생성 방지 로직이 자동으로 처리합니다.

## 운영 중 발견된 사항 — 다음 단계 제안

주간 리포트 실 데이터 첫 생성에서 중요한 것을 하나 발견했습니다.

**소스 `한겨레`가 210건 기사에 평균 중요도 0.9점으로 최대 노이즈 소스로 자동 적발되었습니다.**

이는 AI 관련 소스로 등록된 피드에서 AI와 무관한 일반 뉴스가 대량으로 들어오고 있다는 뜻입니다. 이로 인해 일자별 강도 평균이 실제보다 낮게 계산되는 희석 효과가 발생합니다.

아래 두 가지 방향 중 하나를 권장드립니다.

### 권장 1 — 빠른 해결: 소스 목록 정리

`config/sources.yaml` 에서 AI 전문 피드가 아닌 소스(예: 종합 언론 RSS)를 `enabled: false` 로 비활성화하거나 제거합니다. 이렇게 하면 다음 주부터 바로 깨끗한 집계가 됩니다.

소스가 실제로 AI 관련 기사를 얼마나 내보내는지 확인하는 명령:

```
python main.py --check-sources
```

### 권장 2 — 장기 해결: 월간 리포트 노이즈 필터 (이슈 #12)

V0.2 이후 계획 중인 월간 리포트(이슈 #12) 설계 시, 누적 노이즈 소스 데이터를 활용해 자동 필터 임계값을 설정하는 방향을 제안합니다. 주간 리포트가 적발한 노이즈 소스 정보가 그대로 월간 리포트의 데이터 품질 개선 재료가 됩니다.
