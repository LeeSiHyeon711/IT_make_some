# 실행 안내 — AI-Morning-Brief V0.1

> 이 문서는 AI-Morning-Brief를 처음 설치하고 매일 자동으로 돌리는 방법을 안내합니다.
> "매일 아침 Discord에 AI 뉴스 브리핑이 와 있는 상태"가 목표입니다.

---

## 준비물 확인

시작 전 아래 세 가지가 있어야 합니다.

| 항목 | 확인 방법 | 없으면 |
|------|-----------|--------|
| Python 3.11 이상 | 터미널에서 `python3 --version` 입력 | [python.org](https://www.python.org/downloads/)에서 설치 |
| Anthropic API 키 | `sk-ant-` 로 시작하는 문자열 | [console.anthropic.com](https://console.anthropic.com)에서 발급 (유료) |
| Discord Webhook URL | Discord 채널 설정 > 연동 > 웹훅에서 생성 | 없으면 Discord 전송을 건너뛰고 로컬 파일만 저장됩니다 (선택 사항) |

> Anthropic API 키가 없어도 파이프라인은 동작합니다. 단, Claude 분석 대신 키워드 기반 기본 요약으로 리포트가 생성됩니다.

## 1단계 — 코드 내려받기

GitHub에서 코드를 받아 작업 폴더로 이동합니다.

```
터미널에서 실행:

cd 코드를_넣을_폴더
git clone https://github.com/LeeSiHyeon711/AI-Morning-Brief.git
cd AI-Morning-Brief
```

> 또는 https://github.com/LeeSiHyeon711/AI-Morning-Brief 에서 ZIP으로 내려받아 압축을 풀어도 됩니다.

---

## 2단계 — 의존성 설치 (처음 한 번만)

파이프라인이 사용하는 파이썬 패키지를 설치합니다.

```
cd AI-Morning-Brief
pip install -r requirements.txt
pip install pytest
```

설치가 끝나면 다음 명령으로 확인합니다.

```
python3 main.py --help
```

옵션 목록이 출력되면 설치 성공입니다.

---

## 3단계 — 비밀 키 설정 (처음 한 번만)

`.env` 파일을 만들어 API 키와 Discord 주소를 입력합니다.

```
cp config/.env.example .env
```

이제 `.env` 파일을 텍스트 편집기로 열어 아래 두 줄을 수정합니다.

```
ANTHROPIC_API_KEY=여기에_실제_API_키를_붙여넣기
DISCORD_WEBHOOK_URL=여기에_Discord_Webhook_URL을_붙여넣기
```

주의 사항:
- 이 파일은 절대 타인에게 공유하거나 GitHub에 올리지 마세요. `.gitignore`에 의해 자동으로 제외됩니다.
- Discord Webhook URL이 없으면 그 줄을 빈 칸으로 두어도 됩니다. 로컬 파일만 저장됩니다.

---

## 4단계 — RSS 소스 설정 (처음 한 번만)

수집할 뉴스 소스 목록 파일을 만듭니다.

```
cp config/sources.example.yaml config/sources.yaml
```

`config/sources.yaml` 파일을 열면 기본으로 7개 소스가 예시로 들어 있습니다. 원하는 소스를 추가하거나 필요 없는 소스는 `enabled: false`로 비활성화할 수 있습니다.

```yaml
sources:
  - name: "Hugging Face Blog"
    url: "https://huggingface.co/blog/feed.xml"
    enabled: true
  - name: "원하지 않는 소스"
    url: "https://example.com/feed"
    enabled: false   # 이렇게 하면 수집하지 않습니다
```

## 5단계 — 소스 접근성 확인 (처음 한 번만)

설정한 RSS 주소가 실제로 연결되는지 확인합니다.

```
python3 main.py --check-sources
```

표 형태로 결과가 출력됩니다. `OK: X`로 표시된 소스는 접속이 안 되는 것이므로 URL을 수정하거나 `enabled: false`로 비활성화하세요. 일부 소스가 안 되어도 나머지 소스로 파이프라인이 계속 동작합니다.

## 6단계 — 파이프라인 테스트 실행

실제 RSS·Claude·Discord 없이 샘플 데이터로 전체 흐름을 확인합니다.

```
python3 main.py --test --no-discord
```

터미널에 `완료: 신규 N건, 모드 fallback, 리포트 reports/날짜/report.md` 같은 메시지와 함께 종료코드 0이 표시되면 정상입니다.

생성된 리포트를 직접 확인하려면:

```
cat reports/$(date +%F)/report.md
```

## 7단계 — 매일 자동 실행 등록 (launchd)

Mac의 launchd를 사용해 매일 새벽 4시 30분에 자동으로 실행하도록 등록합니다.

> **launchd를 사용하는 이유**: Mac이 4시 30분에 절전(슬립) 상태여도 깨어난 직후 실행합니다. 기존의 cron은 절전 중에 시간을 놓치면 그냥 건너뛰어 브리핑이 누락됩니다.

### 7-1. plist 파일에 실제 경로 입력

`scripts/com.itsangsang.morningbrief.plist` 파일을 텍스트 편집기로 열어 `/ABS/PATH/TO/`를 실제 절대경로로 교체합니다.

현재 절대경로 확인:

```
pwd
# 예: /Users/홍길동/AI-Morning-Brief
```

수정해야 할 곳 두 군데:
- `main.py` 경로 예시: `/Users/홍길동/AI-Morning-Brief/main.py`
- `WorkingDirectory` 예시: `/Users/홍길동/AI-Morning-Brief`

파이썬3 경로가 기본(`/usr/bin/python3`)과 다르다면 `which python3` 명령으로 확인 후 교체하세요.

### 7-2. 등록

```
cp scripts/com.itsangsang.morningbrief.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.itsangsang.morningbrief.plist
```

### 7-3. 등록 확인 및 즉시 테스트

```
# 등록 확인
launchctl list | grep morningbrief

# 지금 당장 실행해보기 (4시 30분을 기다리지 않고)
launchctl start com.itsangsang.morningbrief

# 실행 로그 확인
cat /tmp/morningbrief.out.log
cat /tmp/morningbrief.err.log
```

### 7-4. 등록 해제 (필요할 때)

```
launchctl unload ~/Library/LaunchAgents/com.itsangsang.morningbrief.plist
rm ~/Library/LaunchAgents/com.itsangsang.morningbrief.plist
```

## Claude 모델 변경 (선택 사항)

기본 모델 대신 다른 Anthropic 모델을 사용하려면 `config/config.yaml`의 `claude.model` 항목을 수정합니다.

```yaml
claude:
  model: claude-opus-4-5   # 원하는 Anthropic 모델명으로 변경
```

모델명이 잘못되거나 존재하지 않으면 자동으로 키워드 기반 기본 요약으로 전환됩니다. 파이프라인은 중단되지 않습니다.

사용 가능한 모델명은 [Anthropic 공식 문서](https://docs.anthropic.com/en/docs/about-claude/models)를 확인하세요.

## 자주 묻는 질문

**Q. Claude 분석이 "AI 분석 실패 / 기본 리포트 생성"으로 나옵니다.**
- `.env` 파일의 `ANTHROPIC_API_KEY`가 올바른지 확인하세요.
- `config/config.yaml`의 `claude.model` 모델명이 유효한지 확인하세요.
- 키가 없거나 모델명이 틀려도 기본 요약으로 리포트가 생성되므로 파이프라인은 중단되지 않습니다.

**Q. Discord 메시지가 오지 않습니다.**
- `.env` 파일의 `DISCORD_WEBHOOK_URL`이 올바른지 확인하세요.
- Discord 전송 실패는 파이프라인을 중단하지 않으며 로컬 리포트는 정상 생성됩니다.

**Q. 특정 RSS 피드가 수집되지 않습니다.**
- `python3 main.py --check-sources` 명령으로 해당 소스의 상태를 확인하세요.
- 접속 불가 소스는 자동으로 건너뛰고 나머지 소스로 계속 진행합니다.

**Q. Mac이 밤에 꺼져 있어서 4시 30분 실행을 놓쳤습니다.**
- 다음 번 Mac이 켜질 때 launchd가 자동으로 실행해 놓친 기간의 기사를 몰아서 수집합니다(catch-up 기능).
- 이때 리포트에는 "보완 수집" 표시와 수집 기간이 명시됩니다.
