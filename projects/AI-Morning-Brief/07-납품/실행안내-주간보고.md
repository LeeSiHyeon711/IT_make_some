# 실행 안내 — AI-Morning-Brief V0.2 주간 흐름 리포트 (`--weekly`)

> 이 문서는 V0.1 일간 브리핑이 이미 운영 중인 상태에서, **주간 흐름 리포트**를 추가로 사용하는 방법을 안내합니다.
> "매주 일요일 Discord에 한 주의 AI 흐름 요약이 와 있는 상태"가 목표입니다.

---

## 주간 리포트란?

일간 브리핑이 "오늘 무슨 일이 있었나?"를 알려준다면, 주간 리포트는 **"이번 주 AI 업계에서 어떤 흐름이 생겼나?"**를 알려줍니다.

7일치 기사 데이터를 종합해 아래 5가지 관점으로 분석합니다.

| 분석 항목 | 쉬운 설명 |
|-----------|-----------|
| 일자별 강도 추이 | 월~토 중 어느 날 AI 뉴스가 가장 많이·크게 터졌는지 |
| 태그 빈도 × 지속성 | "LLM", "에이전트" 같은 키워드가 며칠 동안 꾸준히 나왔는지, 단발로 터졌다 사라졌는지 |
| 소스 신호 품질 | 수집 소스 중 AI와 관련 없는 기사를 많이 뿌린 곳(노이즈 소스)을 자동으로 찾아냄 |
| 공방 관련 픽 | 7일치 기사 중 IT상상공방과 관련도 7점 이상인 기사만 추림 |
| 상상공방 아이디어 통합 | 일간 브리핑에서 각각 나왔던 "오늘 실험해볼 것" 제안 7개를 하나로 통합 |

Claude가 이 다섯 층을 토대로 "이번 주의 흐름"을 서술합니다. Discord에는 9개 블록으로 나뉜 요약 다이제스트가 전송되고, 전문은 파일로 저장됩니다.

---

## 지금 바로 주간 리포트 생성하기 (수동 실행)

이미 V0.1 운영 환경(`~/AI-Morning-Brief-run/`)이 설정되어 있다면 아래 명령 하나로 주간 리포트를 바로 만들 수 있습니다.

### 가장 간단한 실행 (현재 주차 자동 산정)

터미널에서 운영 폴더로 이동한 뒤 실행합니다.

```
cd ~/AI-Morning-Brief-run
python main.py --weekly
```

Discord에 9블록 다이제스트가 전송되고, 전문이 `reports/2026/weekly/W##.md` 에 저장됩니다.

### 명령어 모음

| 상황 | 실행 명령 |
|------|-----------|
| 이번 주 리포트 생성 + Discord 전송 | `python main.py --weekly` |
| 특정 주차 강제 재생성 (예: 26주차) | `python main.py --weekly --week 2026-W26` |
| Discord 전송 없이 파일만 저장 | `python main.py --weekly --no-discord` |
| API 없이 미리보기 (실제 호출 없음) | `python main.py --weekly --force-fallback` |

> 파이썬 경로가 `python` 이 아닌 경우 아래처럼 절대경로로 입력하세요.
> `/Library/Frameworks/Python.framework/Versions/3.14/bin/python3 main.py --weekly`

## 저장되는 파일 위치

주간 리포트 전문은 아래 위치에 자동으로 저장됩니다.

```
~/AI-Morning-Brief-run/reports/
└── 2026/
    └── weekly/
        ├── W26.md    ← 2026년 26주차 전문
        ├── W27.md
        └── ...
```

파인더에서 `홈 폴더 > AI-Morning-Brief-run > reports > 연도 > weekly` 경로로 찾을 수 있습니다.

## 일요일 자동 발화 켜기 (아직 비활성 상태)

현재 일요일 새벽 4시 40분 자동 실행은 **아직 켜져 있지 않습니다.** 아래 절차를 따르면 활성화됩니다.

> 이 절차는 처음 한 번만 합니다. 이후에는 Mac이 알아서 매주 일요일에 실행합니다.

### 1단계 — plist 파일에 실제 경로 입력

`scripts/com.itsangsang.morningbrief.weekly.plist` 파일을 텍스트 편집기로 열어 `/ABS/PATH/TO` 로 표시된 세 곳을 실제 경로로 교체합니다.

교체해야 할 값은 아래와 같습니다.

| 교체 대상 | 바꿀 값 |
|-----------|---------|
| 파이썬 실행 경로 | `/Library/Frameworks/Python.framework/Versions/3.14/bin/python3` |
| `main.py` 경로 | `/Users/여기에_내_사용자명/AI-Morning-Brief-run/main.py` |
| `WorkingDirectory` | `/Users/여기에_내_사용자명/AI-Morning-Brief-run` |

내 사용자명 확인하는 방법: 터미널에서 `echo $HOME` 을 입력하면 `/Users/사용자명` 형태로 나옵니다.

### 2단계 — LaunchAgents 폴더에 등록

터미널에서 아래 명령을 순서대로 실행합니다.

```
cd ~/AI-Morning-Brief-run

cp scripts/com.itsangsang.morningbrief.weekly.plist ~/Library/LaunchAgents/

launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.itsangsang.morningbrief.weekly.plist

launchctl enable gui/$(id -u)/com.itsangsang.morningbrief.weekly
```

### 3단계 — 등록 확인 및 즉시 테스트

등록이 잘 됐는지 확인합니다.

```
launchctl print gui/$(id -u)/com.itsangsang.morningbrief.weekly | grep -iE "state|program|working"
```

일요일을 기다리지 않고 지금 바로 한 번 돌려볼 수 있습니다.

```
launchctl kickstart -k gui/$(id -u)/com.itsangsang.morningbrief.weekly
```

잠시 후 로그를 확인합니다.

```
cat /tmp/morningbrief.weekly.out.log
cat /tmp/morningbrief.weekly.err.log
```

`out.log` 에 "주간 리포트 완료" 또는 "신규 N건" 같은 메시지가 나오고, `err.log` 가 비어 있으면 정상입니다.

### 등록 해제 (필요할 때)

```
launchctl bootout gui/$(id -u)/com.itsangsang.morningbrief.weekly
rm ~/Library/LaunchAgents/com.itsangsang.morningbrief.weekly.plist
```

## 일간과 주간 자동화 타임라인

현재 두 잡이 모두 활성화되면 매주 일요일 새벽은 아래 순서로 흐릅니다.

```
일요일 04:30  → 일간 브리핑 잡 실행 (토요일 기사 수집 + Discord 전송)
일요일 04:40  → 주간 리포트 잡 실행 (직전 7일 집계 + Discord 전송)
```

> 일간 잡이 먼저 토요일 마지막 데이터를 저장하고 나서 주간 잡이 읽기 때문에 10분 간격을 둡니다.
> 두 잡은 서로 독립적으로 동작하므로 한 쪽이 실패해도 다른 쪽에 영향을 주지 않습니다.

## 중복 생성 방지 (자동 처리)

같은 주에 잡이 두 번 실행되더라도(Mac 재시작 등으로 인해) 주간 리포트가 중복 생성되지 않습니다. 내부적으로 마지막으로 생성된 주차(`meta.last_weekly_iso_week`)를 기록해 같은 주 재실행 시 건너뜁니다.

`--week 2026-W26` 처럼 주차를 명시하면 이 방지 로직을 우회해 강제 재생성할 수 있습니다.

## 자주 묻는 질문

**Q. 주간 리포트에 이번 주 기사가 빠진 것 같습니다.**

- 수집된 기사가 DB에 없으면 해당 날짜는 집계에서 제외됩니다. 일간 브리핑이 정상 실행됐는지 먼저 확인하세요.
- `python main.py --weekly --no-discord` 로 Discord 전송 없이 파일만 보고 내용을 확인할 수 있습니다.

**Q. Discord에 주간 메시지가 일간과 같은 채널에 옵니다. 분리할 수 있나요?**

- 현재 V0.2에서는 일간과 주간이 같은 Discord 웹훅을 사용합니다. 채널을 분리하려면 `.env` 에 별도 `DISCORD_WEEKLY_WEBHOOK_URL` 를 추가하는 방식으로 확장 가능하나, 이 기능은 이번 버전 범위 밖입니다. 필요하시면 새 이슈로 접수해 주세요.

**Q. 운영 배포본(`~/AI-Morning-Brief-run`)을 최신 코드로 업데이트하려면?**

```
rsync -a \
  --exclude='.env' --exclude='config/sources.yaml' \
  --exclude='data' --exclude='reports' \
  "/Users/lsh/Desktop/IT_make_some/projects/AI-Morning-Brief/05-개발/" \
  ~/AI-Morning-Brief-run/
```

운영 설정(`.env`, `sources.yaml`)과 데이터(`data/`, `reports/`)는 보존하면서 코드만 갱신합니다.
