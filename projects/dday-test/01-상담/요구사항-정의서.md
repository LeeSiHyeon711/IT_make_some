# 요구사항 정의서 — dday-test

- 작성일: 2026-06-14
- 고객/요청자: 개인 (gyuho1953@gmail.com)

## 1. 무엇을 만드는가

목표 날짜를 입력하면 오늘로부터 남은 일수 또는 지난 일수를 D-7, D-0, D+3 형식으로 즉시 표시하는 단일 D-Day 카운터.

## 2. 누가 쓰는가 (대상 사용자)

- 주 사용자: 고객 본인 (개인용)
- 사용 상황: 시험일·마감일 등 특정 날짜까지 남은 기간을 수시로 확인하고 싶을 때 직접 접속해 사용

## 3. 왜 필요한가 (문제 / 목적)

시험이나 마감일까지 며칠 남았는지 매번 직접 계산하는 번거로움을 없애고, 날짜를 입력하는 것만으로 즉시 D-Day를 확인하기 위함.

## 4. 성공 기준

1. 날짜를 입력하면 오늘 기준으로 남은 일수(예: D-7) 또는 지난 일수(예: D+3)가 화면에 표시된다.
2. 당일(D-0)도 올바르게 표시된다.
3. 한 화면 안에서 입력과 결과 확인이 모두 완결된다.

## 5. 핵심 요구사항 (해야 하는 것)

- [ ] 사용자가 날짜(연·월·일)를 입력할 수 있다.
- [ ] 입력한 날짜와 오늘 날짜를 비교해 D-Day를 계산한다.
- [ ] 결과를 D-숫자(미래) / D-0(당일) / D+숫자(과거) 형식으로 표시한다.
- [ ] 한 화면(단일 페이지)으로 구성된다.

## 6. 제약 / 전제

- D-Day는 한 번에 하나만 관리한다. (다중 항목 목록 불필요)
- 입력한 날짜를 별도로 저장할 필요 없다. 매번 새로 입력해도 무방하다. (브라우저 저장·서버 저장 불필요)
- 고객은 결과물 형태로 '웹페이지'를 명시적으로 요청했다. (기록으로 남기며, 최종 형태 결정은 3단계 설계 몫)

## 7. 미확정 사항 (다음 단계로 넘기는 질문)

- 날짜 입력 방식: 달력(날짜 선택기) UI를 쓸지, 직접 텍스트 입력을 쓸지 — 기획/설계에서 결정.
- D-Day 레이블: "D-7" 앞에 이벤트 이름(예: "토익시험")을 함께 표시할지 — 기획에서 결정.
- 오늘 날짜 기준 처리: 자정(00:00) 기준으로 일수를 계산할지 — 설계에서 결정.
