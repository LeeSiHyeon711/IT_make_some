## 이슈 상세 요약

### #1 페이지 골격 작성
- 구현 범위: `index.html` 마크업 — 제목, 이벤트명 텍스트 입력, 날짜 입력(`input type=date`), 결과 표시 영역, CSS/JS 링크 태그
- 완료 조건: 브라우저에서 열었을 때 모든 영역 렌더링, 설계서 5장 스케치와 구성 요소 일치

### #2 스타일 / 반응형
- 구현 범위: `style.css` — 세로 1열 레이아웃, 결과 강조, 모바일(~375px) 반응형
- 완료 조건: 모바일에서 스크롤 없이 한눈에 보임, 결과 영역 시각적 강조, 데스크톱 레이아웃 정상

### #3 D-Day 계산 로직
- 구현 범위: `calcDday(targetDate, today)` — 로컬 연·월·일 자정 정규화 후 일수 차 정수 반환
- 완료 조건: 타임존 오차 없이 정확한 정수 반환, 당일 0 반환

### #4 결과 포맷 로직
- 구현 범위: `formatLabel(diff, eventName)` — 미래 D-n / 당일 D-0 / 과거 D+n 분기, 이벤트명 접두 처리
- 완료 조건: 3가지 케이스 형식 정확, 이벤트명 유무에 따른 접두 처리 정상

### #5 즉시 갱신 바인딩
- 구현 범위: `render()` + input/change 이벤트 바인딩 — 버튼 없이 실시간 갱신, 미입력 안내 문구
- 완료 조건: 날짜 선택 즉시 결과 갱신, 이름 타이핑 시 실시간 반영, 날짜 삭제 시 안내 문구 복귀

### #6 시나리오 동작 확인 및 빌드노트 작성
- 구현 범위: PRD 4장 3개 시나리오(미래/당일/과거) 수동 점검 + `빌드노트.md` 작성
- 완료 조건: 3개 시나리오 기대 결과 일치 확인, 빌드노트에 실행 방법 및 점검 결과 기록
