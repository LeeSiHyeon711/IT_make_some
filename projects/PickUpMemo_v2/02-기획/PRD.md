## 1. 개요

배민커넥트(`com.woowahan.bros`) 라이더가 신규 배차 카드를 확인하는 순간, Android 접근성 서비스가 화면에서 업체명을 자동 추출하고 사용자가 미리 등록한 메모(주차 안내, 매장 진입법, 특이사항 등)와 일치하면 팝업으로 즉시 표시해주는 Android 앱 MVP다.

라이더는 앱을 전환하거나 별도 조작 없이 배차 판단 순간에 관련 메모를 확인할 수 있다. 서버·OCR·AI 없이 기기 온디바이스로만 동작한다. v1에서 기술 패턴 검증이 완료된 기반 위에 실사용 가능한 MVP를 구현한다.

## 3. MVP 스코프 (이번 사이클)

### 포함 (P0 전체 — F-01 ~ F-13)

- 접근성 서비스 백그라운드 상시 실행 (F-01)
- 신규 배차 카드 화면에서 업체명 자동 추출 (F-02)
- 등록 메모와 `contains` 매칭, 30초 중복 억제 (F-03, F-04)
- 매칭 메모 오버레이 팝업 + 5~8초 자동 닫힘 (F-05, F-06)
- 노이즈 패키지 완전 차단 (F-07)
- 메모 CRUD 전체 (F-08, F-09, F-10, F-11)
- 접근성 권한 안내 화면 (F-12)
- 오버레이("다른 앱 위에 표시") 권한 안내 화면 (F-13)

### 제외 (이번 MVP에서 구현하지 않음)

| 제외 항목 | 이유 |
|-----------|------|
| 배차 수락 후 상세 화면 `가게전화` 패턴 감지 (F-EX-01) | 고객 게이트 확정 결정 ①: 신규 배차 카드 시점만으로 한정. 기술적으로 가능하나 v2 MVP 범위 밖 |
| 타 배달 플랫폼 확장 (F-EX-02) | 고객 게이트 확정 결정 ⑤: 배민커넥트 단독 |
| 메모 내보내기·가져오기 (F-EX-03) | MVP 검증 범위 초과 |
| 태그 기반 검색·필터 (F-EX-04) | 메모 수가 적은 초기 MVP에서 불필요 |

## 4. 핵심 사용자 흐름 (시나리오)

### 시나리오 A — 최초 설정 (앱 설치 후 첫 실행)

1. 사용자가 PickUpMemo_v2 앱을 처음 실행한다.
2. 앱이 접근성 서비스 권한 필요성을 설명하고, Android 접근성 설정 화면으로 이동 버튼을 제공한다.
3. 사용자가 권한을 허용한다.
4. 사용자가 메모 관리 화면으로 이동해 첫 번째 메모를 등록한다. (예: 상호명 "푸라닭", 지점명 "신림점", 메모내용 "지하 주차장 진입 후 엘리베이터 이용", 태그 "주차")
5. 메모 목록에 등록된 항목이 표시됨을 확인한다.
6. 앱을 백그라운드로 내리고 배민커넥트를 실행한다.

### 시나리오 B — 실시간 배차 감지 및 팝업 표시 (핵심 가치)

1. 사용자가 배민커넥트에서 신규 배차 카드 화면을 확인한다.
2. PickUpMemo_v2의 접근성 서비스가 `신규배차_카드` 노드 내에서 `픽업지`~`전달지` 사이의 텍스트("푸라닭 신림점")를 추출한다.
3. 추출된 텍스트가 등록 메모의 상호명("푸라닭") + 지점명("신림점")과 `contains` 매칭으로 일치한다.
4. 배민커넥트 화면 위에 오버레이 팝업이 즉시 표시된다. 팝업에는 상호명·지점명·메모내용·태그가 포함된다.
5. 사용자가 아무 조작 없이 5~8초 후 팝업이 자동으로 사라진다.
6. 같은 메모(동일 메모 ID)에 대해 마지막 팝업 표시로부터 30초 이내에 다시 감지되어도 팝업이 재표시되지 않는다.

### 시나리오 C — 메모 관리 (수정·삭제)

1. 사용자가 PickUpMemo_v2 앱을 열고 메모 목록을 확인한다.
2. 기존 메모를 선택해 수정 화면으로 진입한다.
3. 메모내용 또는 태그를 변경하고 저장한다.
4. 목록으로 돌아와 수정된 내용이 반영됨을 확인한다.
5. 더 이상 필요 없는 메모를 삭제한다.

## 4-1. 메모 매칭 규칙 상세 (F-03)

상호명만 일치하는 오탐을 막기 위해 매칭 성공 조건을 다음과 같이 명확히 한다.

**매칭 성공 조건**
```
detectedStoreText.contains(storeName) AND detectedStoreText.contains(branchName)
```
감지된 업체명 후보 텍스트 안에 등록 메모의 **상호명과 지점명이 모두 포함**되어야 매칭 성공(강한 매칭)으로 판단한다.

- 상호명만 일치하면 → **매칭 실패**
- 지점명만 일치하면 → **매칭 실패**
- 상호명과 지점명이 **모두 포함될 때만** → 강한 매칭(팝업 표시)

**예시**

| 등록 메모(상호명/지점명) | 감지 텍스트 | 결과 |
|---|---|---|
| 푸라닭 / 신림점 | 푸라닭 신림점 | 매칭 성공 |
| 푸라닭 / 봉천점 | 푸라닭 신림점 | 매칭 실패 (지점명 불일치) |
| 푸라닭 / 신림점 | 푸라닭 봉천점 | 매칭 실패 (지점명 불일치) |

## 5. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 백그라운드 동작 | 접근성 서비스는 Android 시스템 정책 범위 내에서 백그라운드 동작을 유지해야 한다. 기기 재부팅 또는 앱 강제 종료 후에는, 사용자가 앱을 다시 실행했을 때 접근성 권한 상태·오버레이 권한 상태·서비스 동작 상태를 확인하고 필요한 경우 재설정 안내를 제공한다. (강제 종료 이후의 무조건 자동 복원은 보장하지 않으며, Android 시스템 정책 범위 내에서 동작한다) |
| 팝업 표시 지연 | 배차 카드 화면 감지 후 팝업 표시까지 체감 지연이 없어야 한다 (수 초 이내 목표) |
| 온디바이스 전용 | 서버 통신·외부 API 호출 없이 완전히 기기 내에서 동작해야 한다 |
| 데이터 저장 | 메모 데이터는 기기 로컬에 저장되며, 앱 재시작 후에도 유지되어야 한다 |
| 오탐 방지 | 제외 패키지 목록에 있는 앱의 접근성 이벤트로 팝업이 발생해서는 안 된다 |
| 팝업 자동 닫힘 | 5초 이상 8초 이하. 정확한 값은 설계 단계에서 결정 (미확정/설계 결정 필요) |

## 7. 수용 기준 (Acceptance Criteria)

MVP 완료 조건. 아래 모두를 충족해야 한다.

| ID | 기준 |
|----|------|
| AC-01 | 배민커넥트 신규 배차 카드 화면에서 "푸라닭 신림점"이 등록된 메모가 팝업으로 표시된다 |
| AC-02 | 팝업은 5~8초 후 자동으로 사라진다. 사용자 터치 없이 |
| AC-03 | 동일 메모 ID에 대해 마지막 팝업 표시 시각으로부터 30초 이내 재감지 시 팝업이 뜨지 않는다 |
| AC-04 | 카카오맵·카카오톡 등 제외 패키지의 화면 이벤트로 팝업이 발생하지 않는다 |
| AC-05 | 등록된 메모가 없는 업체의 배차 카드에서는 팝업이 뜨지 않는다 |
| AC-06 | 앱 내에서 메모를 추가·조회·수정·삭제할 수 있다 |
| AC-07 | 메모 등록 시 상호명·지점명·메모내용 중 하나라도 빈 값이면 저장이 거부된다 |
| AC-08 | 앱이 백그라운드에 있을 때도 감지·팝업이 동작한다 |
| AC-09 | 접근성 권한이 없는 상태에서 앱을 실행하면 권한 안내 화면이 표시된다 |
| AC-10 | 오버레이 표시 권한이 필요한 Android 환경에서 권한이 없을 경우, 앱은 사용자에게 권한 필요성을 안내하고 설정 화면 이동 버튼을 제공한다 |
| AC-11 | 태그가 없는 메모가 팝업으로 표시될 때 태그 영역이 노출되지 않으며, null·빈 문자열·"태그 없음" 같은 문구가 보이지 않는다 |

### 7-1. 매칭 테스트 케이스 (성공·실패)

상호명만 일치하는 오탐을 막기 위해, 성공 케이스뿐 아니라 실패 케이스도 검증 기준에 포함한다.

| 구분 | 등록 메모(상호명/지점명) | 감지 텍스트 | 기대 결과 |
|---|---|---|---|
| 성공 | 푸라닭 / 신림점 | 푸라닭 신림점 | 팝업 표시 |
| 실패 | 푸라닭 / 봉천점 | 푸라닭 신림점 | 팝업 미표시 (지점명 불일치) |
| 실패 | 푸라닭 / 신림점 | 푸라닭 봉천점 | 팝업 미표시 (지점명 불일치) |

## 8. 미확정 / 설계 결정 필요 항목

기획 단계에서 결정하지 않으며, 3단계(설계자)가 판단한다.

| 항목 | 내용 |
