# 요구사항 정의서 — PickUpMemo_v2

- 작성일: 2026-06-17
- 고객/요청자: 배민커넥트 라이더 (v1 검증 완료 후 v2 의뢰)

---

## 1. 무엇을 만드는가

배민커넥트 배차 화면에 등장하는 업체명을 Android 접근성 로그에서 실시간으로 추출하여, 사용자가 미리 등록한 메모(상호명·지점명·메모내용·태그)와 일치할 때 화면에 팝업으로 표시해주는 Android 앱 MVP.
OCR, 외부 서버, AI 없이 동작한다.

---

## 2. 누가 쓰는가 (대상 사용자)

**주 사용자**: 배민커넥트 배달 라이더 (1인)

**사용 상황**:
- 배민커넥트 앱으로 배차 알림을 받고, 신규 배차 카드를 확인하는 순간
- 배차를 수락한 뒤 픽업지 상세 화면을 보면서 출발 준비를 하는 순간
- 두 시점 모두에서 업체 상호가 접근성 로그에 노출되는 것이 v1에서 확인되었다

**전제 환경**:
- Android 스마트폰 (v1 검증 기기: LG)
- 배민커넥트(`com.woowahan.bros`) 앱 설치 및 운행 중
- PickUpMemo_v2 앱은 백그라운드에서 상시 실행 중

---

## 3. 왜 필요한가 (문제 / 목적)

배달 라이더는 자주 가는 업체에 대해 개인 메모(주차 방법, 매장 진입 안내, 특이사항 등)를 갖고 있다. 현재는 배차 수락 여부를 판단하거나 픽업 준비를 할 때 이 메모를 확인하려면 별도 앱을 열거나 기억에 의존해야 한다.

PickUpMemo_v2는 배차 화면에서 업체명이 감지되는 순간 자동으로 해당 메모를 팝업으로 띄워, 라이더가 앱을 전환하지 않고도 즉시 참고 정보를 확인할 수 있게 한다.

v1 기술 검증 결과, 배민커넥트 신규 배차 화면에서 아래 패턴으로 업체명이 접근성 로그에 명확하게 노출됨을 확인했다:

```
신규 배차 카드: 픽업지 / 푸라닭 신림점 / 전달지
배차 수락 후 상세: 가게전화 / 푸라닭 신림점 / 신림동 1433-92 (신림동)
```

---

## 4. 성공 기준

1. **업체명 자동 감지**: 배민커넥트 신규 배차 화면에서 업체명이 접근성 로그로 감지되면, 등록된 메모가 팝업으로 표시된다. 사용자가 다른 앱을 열거나 별도 조작을 하지 않아도 된다.
2. **중복 팝업 없음**: 동일 업체에 대해 30초 이내에 팝업이 중복으로 뜨지 않는다.
3. **오탐 없음**: 배민커넥트(`com.woowahan.bros`) 외 앱(카카오맵, 카카오톡, System UI 등)의 로그로 인해 팝업이 잘못 발생하지 않는다.
4. **메모 저장·조회 동작**: 앱 내에서 등록한 메모(상호명+지점명+메모내용+태그)가 실제 배차 화면 감지 후 팝업에 표시된다.

## 5. 핵심 요구사항 (해야 하는 것)

### 5-1. 업체명 감지 (핵심 기능)

- [ ] Android 접근성 서비스를 통해 `com.woowahan.bros` 패키지의 화면 변경 이벤트를 실시간으로 수신한다
- [ ] 신규 배차 카드 패턴 감지: 로그 내 `id=신규배차_카드` 또는 `픽업지` 키워드를 확인하고, `픽업지`와 `전달지` 사이의 텍스트를 업체명 후보로 추출한다
- [ ] 추출된 업체명 후보에 대해 사용자가 등록한 메모의 상호명·지점명과 `contains` 방식으로 매칭한다
- [ ] 동일 업체명에 대해 30초 이내 중복 감지는 팝업을 발생시키지 않는다

> **MVP 범위 제외 (명시)**: 배차 수락 후 상세 화면 패턴 감지(`가게전화` 다음 텍스트를 업체명으로 추출하는 방식)는 **v2 MVP 필수 범위에서 제외한다.** 해당 패턴은 v1에서 수집 가능함이 확인되었으나, 이번 MVP는 신규 배차 카드 시점 단독으로 한정한다. 이 패턴은 **향후 확장 후보로 문서에만 남기고 구현하지 않는다.**

### 5-2. 메모 팝업 표시

- [ ] 매칭된 메모가 있을 때 현재 화면 위에 팝업으로 메모 내용(상호명·지점명·메모내용·태그)을 표시한다
- [ ] 팝업은 배민커넥트 앱 화면 위에 오버레이로 표시된다 (앱 전환 불필요)
- [ ] 매칭된 메모가 없으면 팝업을 띄우지 않는다

### 5-3. 메모 관리 (앱 내 UI)

- [ ] 사용자가 앱 내에서 메모를 등록할 수 있다. 입력 항목: **상호명(필수), 지점명(필수), 메모내용(필수), 태그(선택)**
- [ ] 등록된 메모 목록을 조회할 수 있다
- [ ] 등록된 메모를 수정·삭제할 수 있다

### 5-4. 노이즈 필터링

- [ ] 메모 매칭 대상은 `com.woowahan.bros` 패키지 로그만으로 제한한다
- [ ] 다음 패키지는 매칭 대상에서 제외한다: `com.android.systemui`, `com.lge.launcher3`, `com.lge.signboard`, `net.daum.android.map`, `com.kakao.talk`, `android` (공유화면), PickUpMemo 앱 자기 자신 패키지

### 5-5. 권한 및 기반

- [ ] Android 접근성 서비스 권한을 앱 내에서 안내하고 설정 화면으로 이동할 수 있다
- [ ] 앱이 백그라운드에서 실행 중일 때도 감지·팝업 기능이 동작한다

## 7. 확정 사항 (고객 게이트에서 결정 완료 — 2026-06-17)

아래 항목들은 상담 게이트에서 고객이 직접 결정한 사항이다. 기획자(2단계)는 이 결정을 PRD의 전제로 삼는다.

| 번호 | 항목 | 고객 확정 결정 |
|---|---|---|
| ① | **팝업 표시 시점** | **신규 배차 카드 감지 시점에만** 팝업을 띄운다. (배차 수락 여부 판단 순간이 가장 중요. 수락 후 상세 화면은 MVP 범위에서 제외.) |
| ② | **팝업 닫힘 방식** | **자동 닫힘**. 일정 시간 후(예: 5~8초) 자동으로 사라진다. 운전·배달 중 화면 조작 부담 제거. |
| ③ | **메모 관리 UI 범위** | **추가·목록 조회·수정·삭제 전부** MVP에 포함(기본 CRUD). |
| ④ | **지점명 처리** | **상호명+지점명 둘 다 필수**. 두 값 모두 있어야 등록·매칭한다. (매칭 정확도 우선.) |
| ⑤ | **대상 플랫폼** | v2 MVP는 **배민커넥트(`com.woowahan.bros`) 단독**. 타 배달 플랫폼 확장은 v2 범위 밖. |

> 참고: ①의 결정에 따라 핵심 요구사항 5-1의 "배차 수락 후 상세 화면 패턴 감지(`가게전화` 패턴)"는 **MVP 필수 범위에서 제외**된다(향후 확장 여지로만 남김). 기획 단계에서 이를 반영한다.
> ④의 결정에 따라 5-3 메모 등록의 지점명은 **선택이 아니라 필수**로 변경된다.
