# 설계서 — PickUpMemo_v2

- 작성일: 2026-06-17

- 참고: `projects/PickUpMemo/PickUpMemo_v1_LogCollector_결과보고서.md`, v1 실제 코드(`projects/PickUpMemo/05-개발/PickUpMemo/`)

## 1. 결과물 형태 결정 ★

- **형태**: **Android 네이티브 앱** (Kotlin).
- **근거**:
  - 핵심 가치(배민커넥트 화면 위 자동 팝업)는 **Android 접근성 서비스 + 시스템 오버레이**가 반드시 필요하다. 웹/자동화로는 타 앱 화면 텍스트를 실시간 수신하거나 그 위에 팝업을 띄울 수 없다.
  - v1(`PickUpMemo`)이 이미 **동작하는 Android 접근성 로그 수집기**다. 검증된 스택·골격을 그대로 재사용하면 "가장 빠르게 확인시킬 수 있는" 결과물이 된다.
  - 서버·OCR·AI 불필요(PRD 5장). 온디바이스 단일 앱이 최소이자 충분한 형태.

| UI | XML View + ViewBinding (Compose 미사용) | v1과 동일. 저가형 모델로 구현 안정성↑, 과한 스택 회피 |

> 과한 스택 금지 원칙 준수: Compose·DI 프레임워크·서버·네트워크 라이브러리 일절 없음. Room만 신규 추가(CRUD 필수).

### 2-1. 포그라운드 서비스 여부 (PRD 설계 결정 항목)
- **포그라운드 서비스 불필요.** `AccessibilityService`는 사용자가 OS 설정에서 권한을 ON으로 둔 동안 **시스템이 직접 바인드·유지**한다(v1에서 검증). 별도 포그라운드 알림이 필요 없다.
- 재부팅/강제종료 후 자동 복원은 보장하지 않는다(PRD 비기능). 대신 `MainActivity.onResume`에서 접근성·오버레이 권한 및 서비스 상태를 재확인하고 안내한다.

### 2-2. 확정한 설계 결정 항목 (PRD 8장)
| 항목 | 확정값 |
|------|--------|
| 팝업 자동 닫힘 초 | **6초 (6000ms)** — 운전 중 읽기 충분 + 화면 과체류 방지(5~8 중간값) |
| 팝업 위치/레이아웃 | **화면 상단** 카드형 오버레이. 배차 카드의 거절/수락 버튼(중앙~하단)을 가리지 않음. 좌우 마진 16dp, 둥근 모서리 카드, elevation, 터치 통과(`FLAG_NOT_TOUCHABLE`) |
| 메모 로컬 저장 | **Room** (SQLite 추상화) |
| 접근성 포그라운드 서비스 | **불필요** (2-1) |
| minSdk / targetSdk | **26 / 34** |
| 앱 패키지명 | **`com.itmakesome.pickupmemo2`** (v1 `com.itmakesome.pickupmemo`와 분리해 동일 기기 공존 가능). 노이즈 제외 목록 + 코드 자기제외에 추가 |

## 4. 화면 / 플로우 스케치

### 4-1. 화면 목록
1. **MainActivity (허브 + 권한 안내)** — LAUNCHER
   - 상단: 앱 제목/한줄 설명
   - 권한 상태 2줄: 접근성 서비스(실행 중/꺼짐), 오버레이 권한(허가/미허가)
   - 버튼: [접근성 설정 열기] [오버레이 권한 설정 열기] [메모 관리]
   - onResume마다 상태 재판정(재부팅/강제종료 후 복귀 시 안내)
2. **MemoListActivity (메모 목록)**
   - RecyclerView: 각 항목 "상호명 지점명" + 메모내용 미리보기(+ 태그)
   - 항목 탭 → 수정 화면, 항목 삭제 버튼, 우하단 [+] → 추가 화면
3. **MemoEditActivity (추가/수정)**
   - 입력: 상호명*, 지점명*, 메모내용*, 태그(선택). [저장] [삭제(수정모드)]
   - 필수 3필드 빈 값이면 저장 거부(AC-07)

### 4-2. 실시간 감지 플로우 (핵심)
```
배민커넥트(com.woowahan.bros) 화면 변경
  → PickupAccessibilityService.onAccessibilityEvent
     ├─ 패키지 == com.woowahan.bros 아니면 무시 (whitelist)
     ├─ 패키지 ∈ EXCLUDED_PACKAGES 면 무시 (자기앱 포함, 방어층)
     ├─ 노드 트리 순회 → 화면 텍스트 조립 (v1 collectNode 재사용)
     ├─ StoreExtractor: "신규배차_카드" 또는 ("픽업지"&"전달지") 확인
     │     → "픽업지"~"전달지" 사이 텍스트 = detectedStoreText
     ├─ MemoMatcher: 모든 메모에 대해
     │     detectedStoreText.contains(storeName) && contains(branchName) → 첫 매칭 반환
     ├─ DedupGuard: 매칭 메모 ID의 마지막 표시로부터 30초 이내면 skip
     └─ MemoPopupController.show(memo): 상단 오버레이, 6초 후 자동 제거
```

### 4-3. 권한 플로우
- 접근성: `Settings.ACTION_ACCESSIBILITY_SETTINGS`로 이동 → 사용자가 PickUpMemo_v2 토글 ON → onResume 재판정
- 오버레이: `Settings.canDrawOverlays` 판정, 미허가 시 `ACTION_MANAGE_OVERLAY_PERMISSION`(package URI)로 이동

### 5-1. 메모 데이터 모델 (Room Entity `Memo`)
| 필드 | 타입 | 설명 |

## 7. v2.1 개선 추가 설계 (실기기 테스트 후 개선요청 3건) ★

- 작성일: 2026-06-17 (3단계 설계자 추가)
- 전제: **FEAT-01~09는 이미 구현·완료**되어 동작 중. 이 절은 사람이 실기기 테스트 후 보고한 3가지 개선요청을 신규 FEAT로 펼친 것이다. 기존 본문(1~6장)·기존 FEAT의 구현·운영 매칭 동작은 보존하며, **추가만** 한다.
- 형태/스택 불변: Android 앱, Kotlin/Room/XML뷰/ViewBinding 재사용. 새 라이브러리·서버·OCR·AI·INTERNET 없음. 내보내기는 로컬 공유 Intent(FileProvider, core-ktx 내장)로만.

### 7-2. 요청 1 — 팝업 검증 경로 (형태·결정)
- **형태**: 앱 내 별도 테스트 화면(`TestActivity`). MainActivity에서 [팝업 테스트]로 진입.
- **결정**: 설계 옵션 (a)+(부분 실파이프라인) 채택, **옵션 (b)는 불채택**.
  - (a) 저장된 메모를 골라 `MemoPopupController.show` 직접 호출 → 팝업 렌더링·오버레이 권한을 즉시 확인.
  - (실파이프라인) 화면 텍스트를 붙여넣어 `StoreExtractor.extract → MemoMatcher.match → 팝업` 을 그대로 태움 → 추출·매칭 로직까지 검증. 이는 **앱 내 입력 텍스트**를 대상으로 하므로 접근성 서비스의 배민 화이트리스트를 건드리지 않는다.
  - (b) "배민 외 패키지도 임시 매칭"하는 디버그 토글은 **운영 매칭 동작을 일시적으로라도 바꾸는 위험**이 있어 채택하지 않는다(안전=기본 동작 불변 우선).
- **기존 FEAT 관계**: FEAT-06(`MemoPopupController`)·FEAT-05(`StoreExtractor`/`MemoMatcher`)·FEAT-02(`MemoRepository`)를 **호출만** 한다. FEAT-07(접근성 서비스)은 **수정하지 않는다.**

### 7-3. 요청 2 — 앱 아이콘 (형태·결정)
- **형태**: adaptive-icon. 외부 에셋 없이 **vector drawable**로 흰 말풍선 + 보라색 메모 줄 심볼.
- **결정**: 배경은 기존 `@color/purple_500` 유지(추가 파일 불필요), foreground만 신규 vector로 교체. minSdk 26이라 `mipmap-anydpi-v26`가 항상 적용되지만 일관성을 위해 `mipmap-hdpi` adaptive-icon XML도 동일 교체.
- **기존 FEAT 관계**: FEAT-01이 만든 mipmap placeholder(전경=흰색)를 대체. colors/themes/매니페스트 아이콘 참조는 불변.

### 7-4. 요청 3 — 배민 알림/접근성 로그 보관함 (형태·결정)
- **형태**: 기존 Room DB(`pickupmemo.db`)에 **테이블 `baemin_logs` 추가**(파일 방식 대신 Room — v2 스택 일관성). 조회는 RecyclerView 화면, 내보내기는 v1 패턴(FileProvider + `ACTION_SEND` txt 공유) 재사용.
- **용량 관리(필수)**: **링버퍼 상한 1000건**(`BaeminLogRepository.MAX_LOGS=1000`). 매 insert 후 `trimTo(1000)`으로 오래된 것 자동 삭제 → 무한 누적 차단. (보관 일수 제한 대신 건수 상한 채택 — 구현 단순·예측 가능)
- **대상 한정**: 접근성 서비스가 이미 `pkg == TARGET_PACKAGE(com.woowahan.bros)` 통과 이벤트만 처리하므로, 그 지점에서 저장하면 **배민 한정**이 자연히 보장된다(다른 패키지 노이즈 미보관).
- **서비스 끼우는 위치**: `PickupAccessibilityService.onAccessibilityEvent`에서 `fullText` 조립 **직후, 기존 extract→match→popup 라인 위**에 `maybeLogBaemin(...)`을 추가한다. 저장은 `serviceScope`(IO)에서 비동기 + 동일 텍스트 3초 중복억제(`LOG_DEDUP_MS`)로 콜백 지연·중복 적재를 막는다. **매칭 로직 라인은 일절 변경하지 않는다(로그 저장은 매칭과 별개의 추가 동작).**
- **마이그레이션 안전**: AppDatabase version 1→2. `fallbackToDestructiveMigration` **금지**(기존 사용자 메모 보존), 대신 `MIGRATION_1_2`(CREATE TABLE baemin_logs)만 적용.
- **기존 FEAT 관계**: FEAT-02(`AppDatabase`/`MemoRepository`)에 마이그레이션·DB 공유 접근자 추가, FEAT-07(`PickupAccessibilityService`)에 저장 결선 추가. 운영 매칭/화이트리스트/노이즈 필터·팝업 동작은 불변.
