## 0. 이 문서의 위치 (v2와의 관계)

v3는 **새 프로젝트가 아니다.** v2 Android 앱(`com.itmakesome.pickupmemo2`) 코드베이스 위에 "픽업지→전달지 자전거 경로 거리/소요시간"을 단일 팝업 안에 추가하는 **확장**이다.

- 형태·스택은 v2에서 확정됨(아래 1장은 계승 명시일 뿐 재결정 아님).
- v2의 3대 기능(V2-1 로그수집 / V2-2 메모 CRUD / V2-3 메모 팝업)은 **회귀 없이 유지**(PRD AC-8, 절대 조건).
- 신규 코드는 기존 패키지 `com.itmakesome.pickupmemo2` 하위에만 추가한다. 새 모듈/새 앱 금지.

### 1-1. 신규 의존성 선택 근거 (과한 스택 회피)
- **HTTP 클라이언트는 OkHttp 1개만 추가.** 카카오 Local + 카카오모빌리티 REST 호출에 표준이며 코루틴과 `withContext(IO)` + 동기 `execute()` 조합으로 단순하게 쓸 수 있다. Retrofit/Moshi/Ktor 등 추가 스택은 응답 1~2종 파싱에 과하다.
- **JSON 파싱은 안드로이드 내장 `org.json`** 사용. 응답 필드가 소수(documents[].x/y, routes[].summary.distance/duration)라 직렬화 라이브러리(Gson/Moshi/kotlinx-serialization)가 불필요하다. 의존성 최소화 원칙(PRD 7장 온디바이스/최소).
- 결론: **신규 라이브러리는 OkHttp 단 1개.** 나머지는 표준 라이브러리 + 코루틴(이미 존재).

## 4. 클래스 설계

### 4-1. route/GeoPoint.kt (FEAT-15)
```kotlin
data class GeoPoint(val lng: Double, val lat: Double)   // 카카오 x=경도(lng), y=위도(lat)
```

### 4-2. route/RouteInfo.kt (FEAT-15)
```kotlin
data class RouteInfo(
    val distanceMeters: Int,
    val durationSeconds: Int,
    val summaryText: String,   // "약 X.Xkm · 약 N분"
    val rawProvider: String    // 예: "kakao"
)
```
- `summaryText` 생성 규칙: km = distanceMeters/1000.0 → `String.format("%.1f", km)`, 분 = `Math.round(durationSeconds/60.0)` (정수). 형식 `약 ${km}km · 약 ${min}분` (PRD 9장).

### 4-3. route/RouteProvider.kt (FEAT-15)
```kotlin
enum class GeoQueryType { ADDRESS, KEYWORD }

interface RouteProvider {
    suspend fun geocode(query: String, type: GeoQueryType): GeoPoint?
    suspend fun getRoute(origin: GeoPoint, dest: GeoPoint): RouteInfo?
}
```
- **설계 결정(인터페이스 시그니처)**: 사용자 합의안의 `geocode(query)` / `getRoute(origin,dest)` 2메서드를 유지하되, `geocode`에 `GeoQueryType` 파라미터를 추가했다. 근거: PRD 4-5/4-6이 **픽업지=키워드 검색**, **전달지=주소검색(4단계) 후 키워드 fallback**으로 검색 엔드포인트가 다르므로, 한 메서드에 검색 타입을 명시하는 편이 구현체(KakaoRouteProvider) 분기를 단순화하고 제공자 교체성을 해치지 않는다.

### 4-4. route/KakaoRouteProvider.kt (FEAT-17)
- 생성자: `class KakaoRouteProvider(private val restApiKey: String) : RouteProvider`
- `geocode(query, ADDRESS)`: `GET https://dapi.kakao.com/v2/local/search/address.json?query=...` → `documents[0].x`(lng)/`y`(lat).
- `geocode(query, KEYWORD)`: `GET https://dapi.kakao.com/v2/local/search/keyword.json?query=...` → 동일 파싱.
- 공통 헤더: `Authorization: KakaoAK $restApiKey`.
- `getRoute(origin, dest)`: 카카오모빌리티 길찾기 `GET https://apis-navi.kakaomobility.com/v1/directions?origin={lng},{lat}&destination={lng},{lat}` (헤더 동일) → `routes[0].summary.distance`(m), `routes[0].summary.duration`(s) → RouteInfo.
- 네트워크는 OkHttp 동기 `execute()`를 `withContext(Dispatchers.IO)`로 감싸 호출. 모든 예외/비정상 응답(빈 documents, result_code≠0 등)은 `null` 반환(throw 금지).
- **엔드포인트 상수는 이 파일 companion에 모은다**(아래 7-3 위험노트의 교체 지점).

### 4-5. route/RouteService.kt (FEAT-18)
```kotlin
object RouteService {
    const val TIMEOUT_MS = 5000L          // ★ 7-1 timeout 정책
    fun init(apiKey: String)              // KakaoRouteProvider 주입(멱등)
    suspend fun resolve(pickup: String, dest: String): RouteInfo?
}
```
- 캐시: `ConcurrentHashMap<String, RouteInfo>` (key = `"$pickup|$dest"`). 동일 배차 반복 조회 비용 절감.
- API 키가 빈 문자열이면 즉시 `null`(자동 fallback — 키 없는 환경에서 크래시 없이 "거리 정보 확인 불가").
- 핵심: 전체(geocode + route)를 `withTimeoutOrNull(TIMEOUT_MS)`로 감싼다(7-1).
- dest는 4단계 geocode fallback(`geocodeDestWithFallback`), pickup은 `geocode(pickup, KEYWORD)`.

### 4-6. matcher/AddressExtractor.kt (FEAT-16)
```kotlin
data class AddressPair(val pickup: String, val dest: String)
object AddressExtractor {
    fun extract(segments: List<String>, fullText: String): AddressPair?
}
```
- **StoreExtractor는 수정하지 않는다.** AddressExtractor는 독립 object.
- 소스 A(우선): `segments` 토큰열에서 `"픽업지"`/`"전달지"`를 포함한 세그먼트 **다음 세그먼트**의 텍스트를 추출.
- 소스 B(fallback): 소스 A로 pickup·dest 중 하나라도 못 얻으면, `"신규배차"`를 포함한 콤마 포함 desc 세그먼트를 콤마 분할해 `"픽업지"`·`"전달지"` 다음 값 추출.
- pickup·dest **둘 다** 확보돼야 `AddressPair` 반환, 아니면 `null`.

### 4-7. overlay/MemoPopupController.kt 확장 (FEAT-19)
기존 `object MemoPopupController` 안에 추가/수정(별도 컨트롤러 금지):
```kotlin
// 신규 토큰 세대
private val generation = java.util.concurrent.atomic.AtomicLong(0)
@Volatile private var currentToken = 0L

// 신규 통합 진입점
fun show(context: Context, memo: Memo?, hasRoute: Boolean): Long
fun updateRoute(token: Long, route: RouteInfo?)

// 기존 시그니처 보존 — 내부 위임(회귀 방어)
fun show(context: Context, memo: Memo) { show(context, memo, hasRoute = false) }
```
- `show(memo, hasRoute)`: `token = generation.incrementAndGet()`를 **post 밖에서** 즉시 만들어 반환. `mainHandler.post{}` 안에서 `currentToken = token`, inflate, 제목/내용/태그 세팅, route 영역 처리:
  - `memo == null` → 제목 칸에 `등록된 메모 없음`(고정 문자열, R.string), 내용/태그 GONE.
  - `hasRoute == true` → route 영역 placeholder(공백/하이픈) VISIBLE → 이후 `updateRoute`로 교체.
  - `hasRoute == false` → route 영역 GONE.
- `updateRoute(token, route)`: `mainHandler.post{}` 안에서 `token != currentToken || currentView == null`이면 무시(stale). 아니면 `tvPopupRoute.text = route?.summaryText ?: "거리 정보 확인 불가"`.
- 자동닫힘(6초)·`dismiss()`·새 show 교체 시 currentToken/currentView가 갱신되므로 늦게 도착한 updateRoute는 자동으로 버려진다(경합 안전).
- 오버레이 권한 미허가 시 기존처럼 표시 skip하되 token은 반환(updateRoute는 currentView null로 무시).

## 5. onAccessibilityEvent 결선 흐름도 (4케이스 + 빌드분기 + 지연표시) ★ (FEAT-21)

기존 라인(보존): `fullText` 조립 → `maybeLogBaemin` → `StoreExtractor.extract` → `MemoMatcher.match` → `DedupGuard.shouldShow(id)` → `MemoPopupController.show`.

v3 결선(승격):
```
onAccessibilityEvent:
  (기존 패키지 필터/화이트리스트/노드조립/maybeLogBaemin 그대로 보존)
  fullText, segments 확보
  maybeLogBaemin(...)                         ← 보존

  // 1) 메모 매칭 (기존 라인 보존)
  candidate = StoreExtractor.extract(fullText)            // null 가능
  matched   = candidate?.let { MemoMatcher.match(it, snapshot) }   // null 가능

  // 2) 주소 추출 (신규)
  addr = AddressExtractor.extract(segments, fullText)     // AddressPair? (null 가능)

  // 3) 트리거 승격 — 둘 다 없으면 종료 (메모만 있어도, 주소만 있어도 진행)
  if (matched == null && addr == null) return
  hasRoute = (addr != null)

  // 4) 중복 억제
  if (matched != null) { if (!DedupGuard.shouldShow(matched.id)) return }
  else                 { if (!DedupGuard.shouldShow(addr!!.key())) return }   // key = "pickup|dest"

  // 5) 케이스 분기
  when {
    matched != null -> {                                   // 케이스 A/B (메모 있음 — 항상 즉시 표시)
       token = MemoPopupController.show(this, matched, hasRoute)
       if (hasRoute) serviceScope.launch {
          val r = RouteService.resolve(addr.pickup, addr.dest)   // 성공:A / 실패:B
          MemoPopupController.updateRoute(token, r)              // r==null → "거리 정보 확인 불가"
       }
    }
    addr != null && BuildConfig.DEBUG -> {                 // 메모 없음 + debug — 즉시 표시(케이스 C/D 모두 표시)
       token = MemoPopupController.show(this, null, hasRoute = true)
       serviceScope.launch {
          val r = RouteService.resolve(addr.pickup, addr.dest)
          MemoPopupController.updateRoute(token, r)              // 실패해도 "거리 정보 확인 불가"(D-debug)
       }
    }
    addr != null -> {                                      // 메모 없음 + release — 지연 표시
       serviceScope.launch {
          val r = RouteService.resolve(addr.pickup, addr.dest)
          if (r != null) {                                       // 케이스 C: 성공 시에만 팝업
             val token = MemoPopupController.show(this, null, hasRoute = true)
             MemoPopupController.updateRoute(token, r)
          }                                                      // 케이스 D-release: 실패 → 미표시(아무것도 안 함)
       }
    }
  }
```

| 케이스 | 메모 | route | debug | release |
|--------|------|-------|-------|---------|
| A | 有 | 성공 | 즉시 팝업 + `약 X.Xkm · 약 N분` | 동일 |
| B | 有 | 실패 | 즉시 팝업 + `거리 정보 확인 불가` | 동일 |
| C | 無 | 성공 | 즉시 팝업(메모없음) → route 표시 | **지연** 팝업(조회 성공 후) + route |
| D | 無 | 실패 | 팝업 표시 + `거리 정보 확인 불가` | **미표시** |

- **스레딩 안전**: AddressExtractor/StoreExtractor/MemoMatcher/DedupGuard/show 호출은 접근성 콜백(메인 스레드)에서 동기. `RouteService.resolve`(네트워크)는 `serviceScope.launch`(IO). `show`/`updateRoute`는 내부에서 `mainHandler.post`하므로 IO에서 호출해도 안전(지연표시 경로 포함).
- **회귀 방어**: 메모 있는 배차(케이스 A/B)는 v2와 동일하게 즉시 메모 팝업이 뜬다(route는 비동기 추가일 뿐). AddressExtractor 추출 실패(addr=null)여도 메모가 있으면 케이스 B로 메모 팝업이 그대로 뜬다 → V2-3 회귀 없음.

## 6. RouteService timeout 정책 (5초) ★ (사용자 필수 지시)

| 항목 | 값 | 근거 |
|------|----|----|
| 배민커넥트 배차 수락 타이머 | 약 **30초** | 라이더가 배차 카드를 보고 수락/거절을 결정할 수 있는 실제 시간. 이 안에 정보가 도움이 되려면 조회는 그보다 훨씬 빨라야 한다 |
| 거리/시간 조회(geocode + route 전체) 최대 대기 | **5초** (`TIMEOUT_MS = 5000`) | 30초 타이머의 1/6. geocode(1~2회) + 경로 1회 왕복을 충분히 커버하면서, 실패 시에도 25초 이상 판단 여유를 남긴다 |
| 5초 초과 시 처리 | `withTimeoutOrNull(5000)` → `null`(실패) | 메인/판단 흐름을 막지 않는다. 무한 대기·ANR 방지 |

**5초 초과(=route null) 후 동작** (5장 케이스와 일치):
- 메모 있음 → 단일 팝업에 `거리 정보 확인 불가` (케이스 B).
- 메모 없음 + release → 팝업 미표시 (케이스 D-release, 지연표시이므로 애초에 팝업이 안 떠 있음).
- 메모 없음 + debug → 디버깅용 팝업 표시 + `거리 정보 확인 불가` (케이스 D-debug).

이 정책은 비기능 요구(앱 크래시 없음·실배달 방해 최소)와 직결된다. 값(5초/30초)은 본 절에 근거와 함께 고정한다.

## 7. 예외 · fallback 설계

### 7-1. 전달지 geocode 4단계 fallback (FEAT-18 RouteService.geocodeDestWithFallback)
| 단계 | 시도 | 메서드 |
|------|------|--------|
| 1 | 원문 그대로 | `geocode(dest, ADDRESS)` |
| 2 | `****` 및 괄호 동명 `(...)` 제거 후 | `geocode(clean, ADDRESS)` |
| 3 | 도로명 앞부분(번지 이하 절단)으로 | `geocode(roadHead, ADDRESS)` |
| 4 | 키워드 검색으로 전환 | `geocode(dest, KEYWORD)` |
- 각 단계 성공(GeoPoint≠null) 시 즉시 반환. 4단계까지 실패면 dest 좌표 null → route null → `거리 정보 확인 불가`.
- 픽업지는 상호/장소명이므로 `geocode(pickup, KEYWORD)` 단일 시도.

### 7-2. 통합 fallback 문구
- 네트워크 실패 / geocode 실패 / 경로 조회 실패 / 5초 timeout → 전부 `RouteService.resolve == null` → route 영역 `거리 정보 확인 불가`. 앱 크래시 없음(PRD 7장).
- API 키 빈 문자열 → resolve 즉시 null(키 미설정 환경에서도 빌드/실행/메모팝업 정상).

### 7-3. ★ 위험 노트 — 카카오 자전거 전용 엔드포인트 불확실성
조사 결과 **카카오모빌리티의 자전거 전용 길찾기 REST 엔드포인트는 공식 제공이 불확실**하다(자동차/도보 위주). 대응:
- v3는 PRD 명시대로 자전거 경로를 1차 시도하되, **엔드포인트는 `KakaoRouteProvider` companion 상수 한 곳에만** 둔다.
- 자전거 엔드포인트가 미지원(404/403)일 경우, **같은 파일의 상수만** 자동차 길찾기(`/v1/directions`)로 교체하면 distance/duration을 그대로 얻을 수 있다(거리값은 유효, 소요시간은 차량 기준이 됨). RouteProvider 추상화 덕에 상위(RouteService/서비스 결선)는 변경 불필요.
- 이는 범위 확대가 아니라 **위험 격리**다. builder는 FEAT-17 범위에서 엔드포인트 상수만 명확히 분리하면 된다.

## 8. API 키 주입 방식 (FEAT-14)

- `local.properties`(git 미추적, Android 기본 .gitignore)에 `KAKAO_REST_API_KEY=...` 한 줄.
- `app/build.gradle.kts`:
  - `android { buildFeatures { viewBinding = true; buildConfig = true } }`
  - `defaultConfig`에서 `local.properties`를 `java.util.Properties`로 읽어 `buildConfigField("String", "KAKAO_REST_API_KEY", "\"$kakaoKey\"")` 주입. 키 없으면 `""`.
- 코드에서는 `BuildConfig.KAKAO_REST_API_KEY` 참조. **소스 하드코딩·git 커밋 금지**(PRD AC-7). RouteService.init에 이 값을 전달.
