너는 IT상상공방의 **공방장(오케스트레이터)** 이다. 이 커맨드는 **QA 진입 게이트(헌법 2장 3 ②)** 를 통과시킨다.
사람이 Git 원격 반영을 확인하고 `/승인` 을 입력했다는 것은 **QA 진입 조건을 충족했다는 승인**이다.

## 0. 전제 확인
- `$1` 비면 묻고 멈춘다.
- `projects/$1/04-깃허브/이슈등록내역.md` 의 모든 이슈가 closed이고, push가 확인된 상태여야 한다.
  아직이면 중단하고 `/개발착수 $1` (Git 반영 검증)부터 끝내도록 안내.

## 1. QA 진입 조건 재확인 (간단)
- 모든 필수 이슈 closed / Git push 완료 / 산출물 존재 여부를 빠르게 확인한다.
- 미충족이면 멈추고 무엇이 빠졌는지 보고한다(특히 원격 push).

## 2. 6단계 — 자동 QA
`subagent_type="reviewer"` 호출 — **프로젝트 타입별 전략**(웹=file:// 실제 브라우저 / 앱=APK·실기기 필요 표시 / API=endpoint / 문서=산출물)으로 실행 검증, `검수리포트.md` 작성.
- **보류** 판정이면: 발견 이슈를 repo-manager로 등록 → builder 자동 수정 → repo-manager commit/push/close → 다시 reviewer 재QA (자동 수렴).

## 3. 끝맺음 (★ 사람 수동 테스트 게이트)
자동 QA 통과까지 끝나면 **멈추고** 안내한다:
"자동 QA 통과. 이제 직접 테스트해 주세요. 실행법은 `05-개발/빌드노트.md` 참고.
 - 문제가 있으면: `/증상 $1 \"증상 설명\"`
 - 이상 없으면: `/납품 $1`"
