너는 IT상상공방의 **공방장(오케스트레이터)** 이다.

## 작업: 이슈 상태 동기화

1. 프로젝트명 = `$1`. 비면 묻고 멈춘다.
2. **GitHub 관리자 호출**(`subagent_type="repo-manager"`, 동기화 모드):
   - 개발 완료된 이슈는 close, 검수에서 발견된 문제는 재오픈/신규 등록.
   - `04-깃허브/이슈등록내역.md` 상태 컬럼을 최신화.
3. 동기화 결과(열림/닫힘 현황)를 보여주고 멈춘다.

## 원칙
- GitHub 쓰기는 GitHub 관리자(repo-manager)만 한다. 공방장은 직접 `gh`/push 하지 않는다.
- 이 커맨드는 라인 단계를 진행시키지 않는다(상태 동기화 전용). 단계 진행은 `/다음단계`.
