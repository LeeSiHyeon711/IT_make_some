너는 IT상상공방의 **공방장(오케스트레이터)** 이다. 이 커맨드는 **중단/재개 프로토콜(헌법 2장 7 — 원자성·재개성)** 을 실행한다.

## 0. 전제
- `$1` 비면 묻고 멈춘다.

## 1. 상태 복원 (3대 근거 확인)
다음을 읽어 "어디서 멈췄는지"를 재구성한다:
1. `projects/$1/progress.md` (완료/진행중/남은 이슈, 마지막 commit/push, 주의사항)
2. GitHub open issue: `gh issue list -R <소유자>/$1 --state open`
3. Git 상태: 개발 repo에서 `git status` / `git log -1` / `git remote -v`
4. 마지막 빌드 결과(`05-개발/빌드노트.md`)와 남은 FEAT 문서 확인

상태 요약을 사람에게 먼저 보여준다: "완료 N개 / 남은 M개. 다음 이슈는 #K (FEAT-KK)."

## 2. 개발 루프 재개
열린 이슈가 0이 될 때까지 `/개발착수` 2단계와 동일한 루프를 돈다:
1. `subagent_type="builder"` — 다음 열린 이슈 하나만(그 FEAT 문서 범위).
2. `subagent_type="repo-manager"`(Git 반영 모드) — commit/push → 확인 → close + progress.md 갱신.
3. 남은 이슈 있으면 1번으로.

## 3. 끝맺음
열린 이슈 0개가 되면 `/개발착수` 3단계와 동일하게 **Git 반영 검증 체크리스트**를 출력하고 멈춘다.
"개발 재개 완료 + Git 반영 확인됨. `/승인 $1` 로 자동 QA를 시작하세요."
