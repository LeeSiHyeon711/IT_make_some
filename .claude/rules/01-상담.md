# 1단계 · 상담 작업 규칙

상위 규칙은 루트 `CLAUDE.md`, 단계 상세는 `docs/pipeline.md` 1단계를 따른다.
이 규칙은 `projects/<프로젝트명>/01-상담/` 폴더에서 일할 때만 적용된다.

## 역할
고객의 막연한 요청을 "확인 가능한 요구사항"으로 구조화한다.

## 입력 / 출력 (계약)
- 입력: 고객의 최초 자연어 요청
- 출력: `01-상담/요구사항-정의서.md` (이 파일 하나만)
- 양식: `templates/요구사항-정의서.md` 를 복제해 채운다

## 반드시 (DO)
- 핵심 4문항을 반드시 확인: **무엇을 / 누가 / 왜 / 성공 기준**.
- 모호하거나 빠진 정보는 **반드시 되묻는다**. 추측으로 채우지 않는다.
- 고객이 "맞다"고 확인할 수 있을 만큼 구체적으로 정리한다.

## 금지 (DON'T)
- 결과물 형태(웹/앱/자동화)를 **단정하지 않는다** — 형태 결정은 3단계(설계) 몫.
- 코드 작성·화면 제작·기술 스택 언급 금지.
- `01-상담/` 폴더 밖에 파일을 쓰지 않는다.

## 완료 조건
요구사항-정의서만으로 기획자가 PRD를 작성할 수 있다. → 사람 승인 후 2단계로.
