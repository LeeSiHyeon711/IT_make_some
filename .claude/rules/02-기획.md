# 2단계 · 기획 작업 규칙

상위 규칙은 루트 `CLAUDE.md`, 단계 상세는 `docs/pipeline.md` 2단계를 따른다.
이 규칙은 `projects/<프로젝트명>/02-기획/` 폴더에서 일할 때만 적용된다.

## 역할
요구사항을 만들 수 있는 제품 명세(PRD)로 구체화한다.

## 입력 / 출력 (계약)
- 입력: `01-상담/요구사항-정의서.md`
- 출력: `02-기획/PRD.md` (이 파일 하나만)
- 양식: `templates/PRD.md` 를 복제해 채운다

## 반드시 (DO)
- 기능을 목록화하고 **우선순위**를 매긴다.
- **MVP 스코프 컷**: "이번에 꼭 보여줄 것" vs "나중 것"을 명확히 분리한다.
- 핵심 사용자 흐름(시나리오) 1~3개를 정의한다.

## 금지 (DON'T)
- 기술 스택·아키텍처·화면 설계로 들어가지 않는다 — 3단계(설계) 몫.
- 코드 작성 금지.
- `02-기획/` 폴더 밖에 파일을 쓰지 않는다. 1단계 산출물을 수정하지 않는다.

## 완료 조건
PRD만으로 설계자가 형태/스택을 결정할 수 있다. → 사람 승인 후 3단계로.
