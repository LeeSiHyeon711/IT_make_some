# 7단계 · 납품 작업 규칙

상위 규칙은 루트 `CLAUDE.md`, 단계 상세는 `docs/architecture/pipeline.md` 7단계를 따른다.
이 규칙은 `projects/<프로젝트명>/07-납품/` 폴더에서 일할 때만 적용된다.

## 역할
데모 실행법과 고객 안내문을 패키징해 고객에게 전달할 수 있게 한다.

## 입력 / 출력 (계약)
- 입력: 전 단계 전체 + 단계별 토큰/시간 기록(`생산성요약.md`)
- 출력: `07-납품/실행안내.md` + `07-납품/고객전달문.md` + `07-납품/증거자료체크리스트.md` + 루트 `생산성요약.md`(최종 요약)
- 양식: `templates/실행안내.md`, `templates/고객전달문.md`, `templates/증거자료체크리스트.md`, `templates/생산성요약.md`

## 반드시 (DO)
- `실행안내.md`: 고객이 결과물을 **직접 보거나 돌리는 법**을 비개발자 눈높이로 적는다.
- `고객전달문.md`: 무엇을 만들었는지 **요약** + 다음 단계(추가 기능/수정) 제안.
- ★ **증거자료체크리스트.md (v2)**: 포트폴리오·회고용 캡처 목록 자동 생성(공정/개발/실행/납품 증거). 타입에 안 맞는 항목은 "해당 없음".
- ★ **생산성요약.md (v2)**: 단계별 토큰/시간 기록을 모아 최종 요약(총 시간·이슈·FEAT·토큰·Builder 평균·최다 토큰 단계·중단여부·승인 게이트 수·실기기 검증). 측정 불가는 "측정 불가"로 정직하게.
- 모든 고객 대면 문장은 비개발자가 이해할 수 있는 언어로 쓴다.

## 금지 (DON'T)
- 코드 수정 금지 (납품은 정리·전달만).
- `07-납품/` 폴더 밖에 새 산출물을 쓰지 않는다(예외: 루트 `생산성요약.md` 최종 요약). 이전 단계 산출물 내용을 수정하지 않는다.
- 토큰/시간에 거짓 수치를 적지 않는다(모르면 "측정 불가").

## 완료 조건
고객이 추가 설명 없이 결과물을 확인할 수 있고, 포트폴리오 증거 목록·생산성 요약이 준비됐다. → 프로젝트 1차 사이클 완료.
(추가 수정 요청은 새 이슈로 등록되어 4→5→6 순환을 다시 돈다.)
