## 기본 프로세스 (7단계 라인)
```
상담가 → 기획자 → 설계자 → Git관리자 → Builder → QA → (사람 수동 테스트) → 인수인계
consultant  planner  architect  repo-manager  builder  reviewer      ▣게이트           delivery
```

| # | 단계 | 담당 | 산출물 | 진행 방식 |
|---|------|------|--------|-----------|
| 1 | 상담 | 상담가(consultant) | 요구사항-정의서 | ▣ 휴먼 게이트 |
| 2 | 기획 | 기획자(planner) | PRD | ▣ 휴먼 게이트 |
| 3 | 설계 | 설계자(architect) | 설계서 + FEAT 문서 | ▣ 휴먼 게이트 |
| 4 | GitHub 관리 | Git관리자(repo-manager) | 이슈등록내역(FEAT 1:1) | ⚙ 자동 |
| 5 | 개발 | Builder(builder) | MVP 코드 + 빌드노트 | ⚙ 자동(이슈 단위 반복) |
| 6 | 검수 | QA(reviewer) | 검수리포트 | ⚙ 자동 |
| — | 수동 테스트 | **사람** | 실사용 확인 | ▣ 휴먼 게이트 |
| 7 | 납품 | 납품담당(delivery) | 실행안내 + 고객전달문 | 게이트 통과 후 |

> 증상/버그가 나오면: **triage(증상분석가)**가 재현·원인 분석 → repo-manager 이슈화 → builder 수정 → reviewer 재검수(자동 수렴). 방향 자체가 틀렸으면 이전 단계로 **되돌리기**.

## 휴먼 게이트가 필요한 지점 (사람만이 줄 수 있는 가치)
1. **방향 결정 게이트** — 상담·기획·설계 (1~3단계). 무엇을/왜 만드는지의 판단.
2. **Git 반영 / QA 진입 게이트** — 모든 개발 이슈 완료 + 코드 원격 반영 확인 후 사람 승인(`/승인`)이 있어야 자동 QA로 진입.
3. **실사용 검증 게이트** — 사람이 직접 수동 테스트를 통과해야 납품으로 진행.

그 사이의 **기계적 실행(이슈 등록·개발·자동 QA·버그 수정)은 게이트 없이 자동**으로 흐른다. 이미 승인된 설계를 옮기는 작업이기 때문이다.

> 공방장 슬래시 커맨드(참고): `/신규프로젝트`, `/다음단계`(상담→기획→설계 한 단계씩), `/개발착수`, `/개발재개`, `/승인`, `/증상`, `/되돌리기`, `/납품`, `/이슈동기화`.

## "바로 구현 금지 — 먼저 PLAN/문서화" 원칙
- 어떤 AI도 **요청을 받자마자 코드부터 짜지 않는다.** 먼저 요구사항·PRD·설계·FEAT로 분해해 문서화한다.
- **FEAT 문서 1개 = 작업 1개 = 이슈 1개.** Builder는 "FEAT 문서 하나만 읽고 그대로 구현 가능"할 만큼 설계가 상세해야 한다.
- 분해·문서화가 끝나고 **사람 승인(게이트)**을 받은 뒤에 구현을 시작한다.
- 한 번의 작업(호출)에서는 **이슈 하나만** 처리하고 상태를 남긴다(끊겨도 재개 가능).

## 버전업 프로젝트 보존 원칙
- 버전업(v2, v3 …)은 **새 프로젝트가 아니다.** 기존 프로젝트의 다음 버전이다.
- **기존 문서와 기존 기능을 보존**한다. 이전 버전 산출물(요구사항/PRD/설계/FEAT/README)을 삭제하거나 덮어쓰지 않는다.
- 새 버전은 **별도 산출물로 추가**하고, 변경점·보존 대상·회귀 대상(기존 기능)을 명시한다.
- 신규 프로젝트와 버전업을 **혼동하지 않는다**(폴더·이슈·repo 분리 명확히). 자세한 금지 항목은 [금지사항](상상공방_금지사항.md) 참조.

## 참고 사례 — PickUpMemo 버전업 (버전업 보존 원칙의 실제 적용)
- v1(로그 수집기) → v2(메모 CRUD + 오버레이 팝업 + 배민 로그 보관함)로 **기능 범위를 넓히며 발전**.
- **PickUpMemo v3(진행 중)**: 실제 사용자 피드백 기반 버전업. v2의 **단일 메모 팝업**에 **픽업지→전달지 경로 거리·예상 소요시간**을 통합(주소 추출·지도 API 조회 신규).
  - 보존 원칙의 모범 예: README가 명시하듯 **v2의 3대 기능(로그 수집 / 메모 CRUD / 메모 팝업)을 회귀 없이 유지**한 채 확장한다. 별도 팝업을 새로 만들지 않고 **같은 팝업의 경로 영역만 비동기 갱신**.
  - 키 분리: 지도 API 키를 **`local.properties`(git 미추적)**에서 주입(코드 하드코딩 없음, 키 없어도 빌드 성공) — [금지사항](상상공방_금지사항.md) 7항 준수.
  - (v3.1) 오르막 주의는 **골격만 선반영**(동작 없음)으로 두고 실제 구현은 다음 버전으로 미룸 — 범위 통제의 예.
