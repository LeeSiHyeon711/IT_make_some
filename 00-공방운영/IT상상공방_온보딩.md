## IT상상공방이란

**고객(또는 본인)의 막연한 아이디어를, 실제로 확인 가능한 MVP로 바꿔 주는 AI 멀티에이전트 생산 공정.**
한 명의 AI가 전부 처리하지 않고, **역할이 분리된 전문 에이전트들이 라인을 따라** 상담 → 기획 → 설계 → 구현 → 검증 → 인수인계를 수행한다. (Claude Code 기반으로 구현되어 있으나, 운영 방식 자체는 도구와 무관한 원칙이다.)

## 핵심 철학 — "상상을 실제 확인 가능한 형태로"
- 말로만 검증하면 어긋난다. **빠르게 만들어 보여주는 것**이 가장 확실한 검증이다.
- 단, 매번 사람이 풀스택으로 만들기엔 느리므로 **반복 가능한 공정**으로 만든다.
- AI에게 무작정 맡기지 않는다. **역할·권한·산출물 계약·검증 기준을 먼저 설계**한다.
- "프롬프트 한 방"이 아니라 **요구사항 분해 · 권한 설계 · 문서 영속화 · 테스트 게이트 · 실사용 피드백 루프**가 본질이다.

## 주요 목적(가치 사슬)
```
아이디어 → 요구사항 정의서 → PRD → 설계서 → FEAT 분해 → 구현 → 검증(QA) → 인수인계
```
각 단계의 산출물이 **다음 단계의 입력 계약**이 된다.

## 주요 산출물
| 단계 | 산출물 |
|------|--------|
| 상담 | 요구사항-정의서 |
| 기획 | PRD |
| 설계 | 설계서 + **FEAT 문서**(작업 1개 = FEAT 1개) |
| GitHub | 이슈등록내역 (FEAT ↔ 이슈 1:1) |
| 개발 | 동작하는 MVP 코드 + 빌드노트 |
| 검수 | 검수리포트(수용 기준·발견 이슈·통과/보류) |
| 납품 | 실행안내 + 고객전달문(인수인계 문서) |

## 현재 디렉토리 구조 (요지)
```
IT_make_some/
├── CLAUDE.md            # 공방 헌법(전 에이전트 공통 규칙)
├── 00-공방운영/         # ← 이 온보딩 문서 세트
├── docs/                # 파이프라인·아키텍처·검증가이드·운영개선(OPS) 문서
├── templates/           # 산출물 템플릿(FEAT·이슈본문 등)
├── .claude/agents/      # 전담 에이전트 8종
├── .claude/commands/    # 공방장 슬래시 커맨드 9종
└── projects/<프로젝트>/ # 01-상담 ~ 07-납품 단계별 산출물 + 05-개발(코드)
```
> 개발 코드(`05-개발`)는 **프로젝트별 자체 git repo로 분리** 보관한다(공방 repo는 공정 산출물 문서만 추적).

## 현재 주요 사례 (참고)
- **PickUpMemo** — Android 앱. 배민커넥트 신규 배차 시 저장된 가게 메모를 오버레이 팝업으로 표시(접근성 서비스 기반 기술검증). v1=로그수집기 → v2=메모 CRUD+팝업+로그 보관함.
- **아이디어 건강검진**(sangsang-lite) — "만들기 전에 먼저 확인"을 돕는 룰 기반 MCP 서버(아이디어→균열점→첫 검증 미션).
- **AI-Morning-Brief** — Python CLI 자동화. RSS 수집→SQLite→Claude 분석/fallback→Markdown 리포트→Discord 전송(V0.1, launchd 04:30 등록). *정기 실전 검증은 대기 중.*
- (그 외) PawprintDiary(반려동물 일기+AI, 로컬 우선 웹), D-Day 카운터(라인 첫 검증 사례).
