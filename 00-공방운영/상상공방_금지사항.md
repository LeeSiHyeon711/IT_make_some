# 상상공방 금지 사항

> 작성일: 2026-06-24
> 문서 목적: 어떤 AI가 어느 역할을 맡든 **절대 하면 안 되는 것**을 모은다. 위반은 곧 신뢰·안전 문제다.

## 문서·버전 관련
1. **기존 문서 삭제 금지.** 산출물(요구사항/PRD/설계/FEAT/README/리포트)을 임의로 지우지 않는다.
2. **버전업 시 이전 버전 문서 덮어쓰기 금지.** v2의 산출물을 v3가 덮지 않는다. 새 버전은 **별도 산출물로 추가**하고 변경점·보존 대상을 명시한다.
3. **신규 프로젝트와 버전업 혼동 금지.** 버전업은 새 프로젝트가 아니다. 폴더·이슈·repo·범위를 분명히 구분한다.

## 프로세스 관련
4. **승인 전 구현 금지.** 요청을 받자마자 코드부터 짜지 않는다. 먼저 요구사항→PRD→설계→FEAT로 분해·문서화하고, 휴먼 게이트(방향 결정) 승인을 받은 뒤 구현한다. ("바로 구현 금지, 먼저 PLAN" — [작업흐름](상상공방_작업흐름.md))
5. **QA 없이 이슈 close 금지.** 검수(수용 기준·회귀)를 통과하지 않은 작업을 완료 처리하지 않는다. (Git 반영 확인 + 사람 `/승인` 전까지 자동 QA 진입도 금지 — 헌법 2장 9)
6. **"좋아 보이는 기능"을 무작정 추가하지 않는다.** 모든 기능은 **사용자 문제와 연결**되어야 한다. 범위 확장은 근거(문제·요구)와 승인을 동반한다.

## 보안·비용 관련
7. **API Key·토큰·비밀번호 하드코딩 금지.** 소스/문서/로그/커밋/대화 출력에 비밀값을 노출하지 않는다.
   - 비밀값은 `.env`(앱별) / `local.properties`(Android) / Secret Manager / 쉘 환경변수에만 두고 **반드시 git에서 제외**한다.
   - 좋은 예: PickUpMemo는 카카오 REST 키를 `local.properties`에서 읽어 `BuildConfig`로 주입(코드 하드코딩 없음, 키 없어도 빌드 성공). AI-Morning-Brief·PawprintDiary는 키를 `.env`/`.env.local`로 분리.
8. **외부 비용이 발생하는 API/LLM 사용 전 반드시 확인.** 유료 모델 호출, 외부 지도/검색 API 등 과금 가능 작업은 실행 전에 사람에게 알리고 승인을 받는다.

## Git·권한 관련
9. **권한 없는 주체의 직접 git 쓰기 금지.** Builder/검수자 등은 직접 push/이슈 close를 하지 않는다(차단). Git 쓰기는 권한 있는 주체(repo-manager / 승인된 공방장·사람)가 수행한다. (현재 기본 운영 = OPS-01 **L1.5**: 읽기/검증 자동, 쓰기는 사람 승인 후 실행 — `docs/operations/OPS-01-git-flow-design.md`)
10. **위험한 git 명령(force push·히스토리 변경·원격 강제 동기화 등)은 항상 사람 승인.** 레벨과 무관하게 차단/강승인 대상이다.

## 정직성
11. **결과를 과장하지 않는다.** 수치는 보수적으로, 미검증은 "대기/예정"으로 표기한다. 구현되지 않은 기능을 "완료"로 적지 않는다.
