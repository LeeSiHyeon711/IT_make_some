# 아이디어 건강검진 (sangsang-lite) — 진행상황

> 작성일: 2026-06-24 · 최종 갱신: 2026-06-24
> 문서 목적: IT상상공방에서 갈라져 나온 경량 MCP 서비스의 진행 과정·이터레이션을 기록한다. 인덱스: [README.md](README.md)
> 한 줄 정의: 막연한 아이디어를 **"만들 것"이 아니라 "먼저 확인할 것"**으로 바꿔주는 룰 기반 MCP 도구. 공방 접수 확인 → **균열점 진단** → **첫 검증 미션**(오늘 할 수 있는 최소 실험)으로 변환.

## 현재 상태 (한 줄)
✅ 코드 완성·푸시(파싱 안정화 + 1회 문진 UX까지). ⚠ **Cloud Run 재배포 + PlayMCP 등록·재테스트는 사용자 액션 대기.**

## 아키텍처(확정)
- **Streamable HTTP / stateless** MCP 서버(FastMCP), 엔드포인트 `/mcp`, 식별자 `sangsang-lite`.
- **3-tool** 구조(룰/템플릿 기반, **서버 내부 LLM 호출 없음** — PlayMCP 평균 100ms·p99 3000ms 요건 충족, p99 약 36ms 실측):
  1. `prepare_intake` — 아이디어 접수·구조화(문진표 생성기)
  2. `diagnose_idea` — 균열점 1개 진단(focus 6종)
  3. `design_first_experiment` — 첫 검증 미션 설계(시간·대상 수 반영)
- 자연어 카드는 PlayMCP **AI채팅 본체**가 다듬는다(서버는 '검증 미션 재료'만 반환).

## 결과카드 구조(현행)
균열점 → 첫 검증 미션 → 미션 단계 → 성공 기준 → 실패 신호 → 아직 만들지 말아야 할 것 → 다음 단계. (점수체계 없음, 틸드(~) 미사용)

---

## 주요 이터레이션 이력 (등록 안정화 과정)
> 명시적 vN 표기는 없으나, PlayMCP 실테스트 피드백으로 반복 개선했다. 시간순 핵심:

1. **서버 내부 LLM 전면 제거** → 룰/템플릿 기반 전환(지연 요건 충족이 불가능했던 LLM 구조 폐기).
2. **입력 방어 + clarification 모델** + rich/원문 target_user 보존 + 조사(josa) 보정.
3. **오케스트레이션 원칙 명시**(아이디어 나오면 prepare_intake 먼저, 임의 질문 금지) + tool description 사용자 친화 축약.
4. **틸드(~) 제거**(마크다운 취소선 깨짐 방지).
5. **검증 준비도 점수 추가 → 다시 제거**(카카오 챗이 점수 세부를 안정적으로 출력 못함 → UI 강제보다 흐름 안정성 우선).
6. **"아이디어 건강검진" 컨셉으로 리브랜딩**(식별자 `sangsang-lite`는 유지).
7. **진단 포커스 6종 다양화**(PAIN_INTENSITY/CONTEXT_OF_USE/WILLINGNESS/ALTERNATIVE_BEHAVIOR/DATA_INPUT_BURDEN/TIMING_URGENCY) — 결과카드가 아이디어마다 달라지게.
8. **WILLINGNESS 기록형 전용 미션** 구체화(운동기록 등).
9. **일반 명사구 파싱 안정화**(주어/목적어/부사 분리, 빈 값·부사 오염 해결).
10. **prepare_intake를 "문진표 생성기"로 전환** — 확신 게이트(확신 낮으면 억지 생성 금지·안전 fallback), **1회 문진**(`has_clarified`), `understood_summary`/`extracted_fields_summary`/`one_shot_clarification_prompt` 제공, 시간·대상 수·대체방법 파싱.
